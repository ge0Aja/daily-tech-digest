_Vercel Breached via Context AI OAuth Supply Chain Attack — Lumma Stealer at Root_

_Bleeping Computer_ | [Read more](https://www.bleepingcomputer.com/news/security/vercel-confirms-breach-as-hackers-claim-to-be-selling-stolen-data/)

A Context.ai employee was infected by Lumma Stealer malware (delivered via fake Roblox scripts) in February, giving attackers their Google Workspace OAuth tokens and a pivot path into Vercel's internal systems. Non-sensitive environment variables were accessed; Vercel confirmed npm packages are clean and engaged Mandiant and law enforcement. A BreachForums actor claiming ShinyHunters affiliation — which the group denies — is advertising stolen API keys, source code, and database data for $2M; the full attack chain represents a textbook OAuth-via-supply-chain intrusion.

_NIST Limits CVE Enrichment After 263% Surge in Vulnerability Submissions_

_Help Net Security_ | [Read more](https://www.helpnetsecurity.com/2026/04/16/nist-national-vulnerability-database-nvd-enrichment/)

NIST shifted the National Vulnerability Database to a risk-based enrichment model on April 15 after CVE submissions surged 263% between 2020 and 2025. Going forward, NIST will only fully analyze CVEs appearing on CISA's KEV catalog, those affecting federal government software, and critical software under EO 14028 — all others will be listed but left unenriched. Security teams relying on NVD for CVSS scores and CPE data on non-critical software must now source that data elsewhere; tooling built on the assumption of universal NVD enrichment will need to be updated.

_Cisco Patches Critical CVE-2026-20184 (CVSS 9.8): Webex SSO Impersonation_

_The Hacker News_ | [Read more](https://thehackernews.com/2026/04/cisco-patches-four-critical-identity.html)

Cisco patched four critical vulnerabilities in Identity Services Engine and Webex Services, the most severe being CVE-2026-20184 (CVSS 9.8) — an improper certificate validation flaw in Webex's SSO integration with Control Hub that allows unauthenticated remote attackers to impersonate any user within the service. The fix requires customers to manually upload a new IdP SAML certificate to Control Hub; it is not auto-deployed. Cisco reports no known in-the-wild exploitation, but with a 9.8 CVSS and no authentication required, organizations running Webex SSO should treat this as high priority.

_SAP CVSS 9.9 SQL Injection (CVE-2026-27681) Leads Critical April Patch Tuesday_

_The Hacker News_ | [Read more](https://thehackernews.com/2026/04/april-patch-tuesday-fixes-critical.html)

April's Patch Tuesday brought a critical SQL injection flaw in SAP Business Planning and Consolidation (CVE-2026-27681, CVSS 9.9) that enables arbitrary database command execution — the highest severity in this cycle. Fortinet and Adobe also shipped high-severity patches, adding to a month where CISA's KEV catalog expanded by six new entries across Fortinet, Microsoft, and Adobe products. Organizations running SAP BPC in particular should treat CVE-2026-27681 as an emergency patch given the 9.9 score and database-level blast radius.

_MySQL 8.0 Reaches End of Life — Over Half of Deployments Still Unpatched_

_Percona_ | [Read more](https://www.percona.com/blog/mysql-8-0-end-of-life-date/)

MySQL 8.0 hits end-of-life on April 30, 2026, ending all security patches, bug fixes, and Oracle support for what remains the most widely deployed MySQL version — over half of production MySQL instances are still running 8.0. The intended upgrade path is MySQL 8.4 LTS (premier support through April 2029), but it is not a drop-in upgrade: `mysql_native_password` is disabled by default in 8.4, requiring an authentication plugin audit before upgrading. Teams should verify plugin compatibility now; instances still on 8.0 after April 30 will accumulate unpatched CVEs with no upstream fix path.
