## Security & Privacy

### nginx-ui CVE-2026-33032 (CVSS 9.8): Full Nginx Takeover Under Mass Exploitation
**Source:** The Hacker News | [Read more](https://thehackernews.com/2026/04/critical-nginx-ui-vulnerability-cve.html)

nginx-ui's MCP integration left the `/mcp_message` endpoint unauthenticated while `/mcp` enforced IP whitelisting — a trivially exploitable authentication split. Two HTTP requests are sufficient: establish an SSE session, then invoke any tool including rewriting Nginx configs, restarting the service, or redirecting traffic. Shodan shows ~2,689 internet-exposed instances; patch to version 2.3.4 immediately.

### SGLang CVE-2026-5760 (CVSS 9.8): RCE via Poisoned GGUF Model Files
**Source:** The Hacker News | [Read more](https://thehackernews.com/2026/04/sglang-cve-2026-5760-cvss-98-enables.html)

A Jinja2 server-side template injection in SGLang's `/v1/rerank` endpoint lets attackers execute arbitrary OS commands when a victim loads a crafted GGUF model file. The malicious `tokenizer.chat_template` contains a trigger phrase that activates Qwen3 reranker logic, escapes the template context, and runs attacker-controlled commands on the host. ML teams must treat all external GGUF files from public repositories as untrusted executables.

**Source:** Bleeping Computer | [Read more](https://www.bleepingcomputer.com/news/security/new-fortinet-forticlient-ems-flaw-cve-2026-35616-exploited-in-attacks/)
