### OpenAI Releases GPT-5.5
**Source:** OpenAI | [Read more](https://openai.com/index/introducing-gpt-5-5/)

OpenAI launched GPT-5.5 on April 23 and opened API access the following day. ### DeepSeek V4 Pro & Flash Released Under MIT License
**Source:** DeepSeek | [Read more](https://api-docs.deepseek.com/news/news260424)

Released April 24, DeepSeek V4 Pro is a 1.6T-parameter MoE model with 49B active parameters achieving 80.6% on SWE-bench Verified — within 0.2 points of Claude Opus 4.6 — and leading on LiveCodeBench (93.5%) and Terminal-Bench 2.0. ## Security & Privacy

### Bitwarden CLI Compromised in Checkmarx Supply Chain Attack
**Source:** Bleeping Computer | [Read more](https://www.bleepingcomputer.com/news/security/bitwarden-cli-npm-package-compromised-to-steal-developer-credentials/)

On April 22, `@bitwarden/cli@2026.4.0` on npm carried a malicious preinstall hook for roughly 90 minutes. The payload stole GitHub/npm tokens, SSH keys, `.env` files, cloud credentials, and shell history — then exfiltrated them encrypted to attacker-controlled GitHub repositories. The compromise traced upstream to Checkmarx's `checkmarx/ast-github-action` GitHub Action being hijacked as part of a broader supply chain campaign. Bitwarden confirmed no vault data was accessed; any team that installed the package during that window should immediately rotate all exposed secrets and audit CI secrets.

### Three Windows Defender Zero-Days Actively Exploited; Two Still Unpatched
**Source:** The Hacker News | [Read more](https://thehackernews.com/2026/04/three-microsoft-defender-zero-days.html)

Researchers disclosed three Windows Defender local privilege escalation vulnerabilities — BlueHammer (CVE-2026-33825), RedSun, and UnDefend — within a 13-day window in April. All three are exploiting TOCTOU race conditions or file-path manipulation in Defender's threat remediation engine to gain SYSTEM-level access on fully patched Windows 10 and 11. Microsoft patched only BlueHammer in April's Patch Tuesday; RedSun and UnDefend remain unpatched with active exploitation observed since April 16. Until Microsoft ships fixes, the practical mitigations are minimal on updated systems.

### LMDeploy SSRF Flaw (CVE-2026-33626) Exploited Within 13 Hours of Disclosure
**Source:** Sysdig / The Hacker News | [Read more](https://thehackernews.com/2026/04/lmdeploy-cve-2026-33626-flaw-exploited.html)

CVE-2026-33626 (CVSS 7.5) is a Server-Side Request Forgery flaw in LMDeploy's vision-language image loader: the `load_image()` function fetches arbitrary URLs without validating internal/private IPs, allowing attackers to reach cloud metadata services (AWS IMDS), internal Redis/MySQL instances, and admin interfaces. Sysdig's honeypot detected the first exploitation attempt 12 hours 31 minutes after GitHub disclosure — in a single 8-minute session, the attacker used the vision endpoint to port-scan an entire internal network and probe OOB DNS exfiltration. Teams running any pre-0.12.0 LMDeploy version with vision-language support should upgrade to v0.12.3 immediately.

### CISA Flags 6 Actively Exploited Flaws: Fortinet SQL Injection, Adobe RCE, Microsoft Exchange
**Source:** CISA | [Read more](https://www.cisa.gov/news-events/alerts/2026/04/13/cisa-adds-seven-known-exploited-vulnerabilities-catalog)

On April 13, CISA added seven CVEs to its Known Exploited Vulnerabilities catalog, requiring FCEB agencies to patch by April 27. Key additions: CVE-2026-21643 (CVSS 9.1) — an unauthenticated SQL injection in Fortinet FortiClient EMS enabling remote code execution; CVE-2026-34621 (CVSS 8.6) — an Adobe Acrobat Reader flaw allowing malicious JavaScript execution via crafted PDFs; and CVE-2023-21529 — a deserialization RCE in Microsoft Exchange Server exploitable by authenticated attackers. The April 27 deadline makes these the highest-priority enterprise patches this week.

## Cloud & Infrastructure

### AWS Trainium3 UltraServers Now Generally Available
**Source:** Amazon Web Services | [Read more](https://www.aboutamazon.com/news/aws/trainium-3-ultraserver-faster-ai-training-lower-cost)

Amazon's EC2 Trn3 UltraServers — built on AWS's first 3nm AI chip — are now GA. Trainium3 delivers 2.52 PFLOPs of FP8 compute (2× Trainium2), 144 GB HBM3e memory at 4.9 TB/s bandwidth, and scales to 144 chips per UltraServer (362 PFLOPs total). With Amazon guiding $200B in AI infrastructure capex for 2026, Trainium3 is central to the company's strategy to reduce dependence on NVIDIA for large-scale model training and inference.

## Industry & Trends

### Intel Shuts Down Open-Source Evangelism Program, Archives 24+ Projects
**Source:** Phoronix | [Read more](https://www.phoronix.com/news/Intel-Ends-OSS-Evangelism-Repos)

Intel formally ended its Open Ecosystem Community and Evangelism initiative in late April, archiving the program's GitHub repository and shutting down dozens of OSS projects — including its Xeon AI time-series forecasting toolkit, High Density Scalable Load Balancer (built on DPVS/DPDK), and the Software Defined Silicon (SDSi) project. The move concludes months of quiet pruning amid restructuring and financial pressure; Intel's last prominent open-source evangelist departed in July 2025. For the broader ecosystem, the retreat signals a shift from broad community leadership — a role Intel played for decades in Linux and toolchain development — to narrower, product-aligned contributions.

### Chainguard and Cursor Partner to Secure the Agentic Coding Supply Chain
**Source:** PR Newswire | [Read more](https://www.prnewswire.com/news-releases/chainguard-and-cursor-partner-to-secure-agentic-coding-with-trusted-open-source-302748112.html)

Chainguard and Cursor announced a partnership on April 21 to harden the open-source supply chain for AI-driven development. Chainguard provides a trust layer for container images and OSS libraries that Cursor's multi-model coding environment pulls and executes, ensuring artifacts are malware-free and secure-by-default. As agentic coding tools autonomously resolve, pull, and run dependencies at scale, supply chain integrity for those artifacts becomes a critical security control — and the Bitwarden CLI incident this week makes the timing particularly apt.
