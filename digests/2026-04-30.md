## Security & Privacy

### CVE-2026-32202: Windows Shell Zero-Click Actively Exploited by APT28
**Source:** The Hacker News | [Read more](https://thehackernews.com/2026/04/microsoft-confirms-active-exploitation.html)

Microsoft and CISA confirmed active exploitation of CVE-2026-32202, a zero-click Windows Shell spoofing vulnerability stemming from an incomplete patch for an earlier APT28/Fancy Bear exploit. Attackers steal a victim's Net-NTLMv2 authentication hash by having the user simply browse to a folder containing a malicious .lnk file — no file opening required. CISA set a May 12 deadline for federal agencies to remediate. Affected systems include Windows 10, 11, and multiple Windows Server versions.

### Critical GitHub RCE CVE-2026-3854: 88% of Self-Hosted Instances Still Unpatched
**Source:** Help Net Security | [Read more](https://www.helpnetsecurity.com/2026/04/29/cve-2026-3854-github-rce-vulnerability/)

Wiz researchers found CVE-2026-3854, a command injection in GitHub's internal babeld git proxy that lets any user with push access achieve remote code execution via a crafted push option with an unsanitized semicolon character. On GitHub Enterprise Server this means full server compromise — access to all hosted repositories and internal secrets. GitHub patched github.com within two hours of the March 4 report, but as of April 28, 88% of self-hosted GHES instances remain unpatched. Upgrade to GHES 3.19.3 or later immediately.

### ADT Breach: 5.5M Customers Hit After ShinyHunters Okta Vishing Attack
**Source:** BleepingComputer | [Read more](https://www.bleepingcomputer.com/news/security/home-security-giant-adt-data-breach-affects-55-million-people/)

ADT confirmed a breach affecting 5.5 million customers after ShinyHunters compromised an employee's Okta SSO account via voice phishing (vishing), then pivoted into ADT's Salesforce instance. ### prt-scan: AI-Powered GitHub Supply Chain Attack Targets 500+ Repositories
**Source:** Wiz Research | [Read more](https://www.wiz.io/blog/six-accounts-one-actor-inside-the-prt-scan-supply-chain-campaign)

A threat actor ran a six-wave campaign exploiting GitHub's pull_request_target workflow trigger, opening 500+ malicious PRs using AI-generated, repository-aware payloads adapting to each target's stack — Go test files for Go repos, npm scripts for JavaScript projects. The campaign successfully exfiltrated secrets from 50+ repositories including AWS keys, Cloudflare tokens, and Netlify credentials. This marks a fundamental shift: AI automation has lowered the barrier to large-scale supply chain attacks beyond what manual code review can defend.

### Trivy GitHub Actions Tags Hijacked in CI/CD Supply Chain Compromise
**Source:** Aqua Security | [Read more](https://www.aquasec.com/blog/trivy-supply-chain-attack-what-you-need-to-know/)

Attackers exploited a misconfigured GitHub Actions environment to extract privileged tokens from the Trivy scanner, then force-pushed malicious code to 76 version tags in the aquasecurity/trivy-action repository. The payload silently exfiltrated API tokens, cloud credentials (AWS/GCP/Azure), SSH keys, Kubernetes tokens, and Docker configs while appearing to complete normal scans. Any CI/CD pipeline that ran compromised Trivy versions should rotate all secrets immediately — security tools themselves are high-value supply chain targets.

### TypeScript Overtakes Python as #1 Language on GitHub
**Source:** GitHub Blog | [Read more](https://github.blog/news-insights/octoverse/octoverse-a-new-developer-joins-github-every-second-as-ai-leads-typescript-to-1/)

TypeScript became the most-used language on GitHub in August 2025, surpassing Python for the first time — the most significant language shift in over a decade. TypeScript reached 2.6 million monthly active contributors, growing 66% year-over-year, driven by AI coding assistants that favor statically-typed languages and major frontend frameworks defaulting to TypeScript scaffolding. Python retains dominance in AI/ML workloads, but the broader developer community has decisively moved to TypeScript for application development.
