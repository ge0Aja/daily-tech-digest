# Tech Digest — May 1, 2026

## AI & Machine Learning

### Yann LeCun's AMI Labs Raises $1.03B to Build World Models
**Source:** TechCrunch | **URL:** https://techcrunch.com/2026/03/09/yann-lecuns-ami-labs-raises-1-03-billion-to-build-world-models/

AMI Labs, co-founded by Turing Award winner Yann LeCun after leaving Meta, raised $1.03B at a $3.5B valuation — the largest seed round in European history. The company is building "world models" based on LeCun's Joint Embedding Predictive Architecture (JEPA), an alternative to LLMs that learns by modeling how the physical world works rather than text patterns. Target applications include industrial robotics and healthcare. Backers include Cathay Innovation, Greycroft, Bezos Expeditions, Eric Schmidt, and Tim Berners-Lee.

## Security & Privacy

### LiteLLM CVE-2026-42208: Critical Pre-Auth SQL Injection Exploited in 36 Hours
**Source:** The Hacker News / BleepingComputer | **URL:** https://thehackernews.com/2026/04/litellm-cve-2026-42208-sql-injection.html

A CVSS 9.3 pre-authentication SQL injection in LiteLLM — an open-source LLM gateway with 22K+ GitHub stars used to front OpenAI, Anthropic, and other providers — was patched April 19 but exploited in the wild just 36 hours after the advisory was indexed. The flaw passes unsanitized Authorization header values directly into raw SQL, letting unauthenticated attackers exfiltrate LLM provider API keys from the PostgreSQL backend. Affected versions: >=1.81.16 and <1.83.7-stable; patch immediately.

### GitHub CVE-2026-3854: RCE via Single git push, 88% of Enterprise Servers Unpatched
**Source:** Wiz / GitHub Blog | **URL:** https://www.wiz.io/blog/github-rce-vulnerability-cve-2026-3854

Wiz discovered a CVSS 8.7 command injection in GitHub's git push pipeline: any user with push access could execute arbitrary commands on GitHub's backend servers using a single crafted push option. On GitHub.com, shared storage nodes hosting millions of public and private repos were accessible; on GitHub Enterprise Server (GHES), full server compromise was possible. GitHub.com was patched within two hours of disclosure; GHES patch (3.19.3+) released March 10 — but 88% of enterprise instances remained unpatched at public disclosure April 28.

### Linux "Copy Fail" CVE-2026-31431: Root Access on Every Major Distro Since 2017
**Source:** CyberSecurityNews / The Register | **URL:** https://cybersecuritynews.com/linux-kernel-0-day-copy-fail/

A 9-year-old logic flaw in the Linux kernel's algif_aead cryptographic module lets any unprivileged local user corrupt page cache entries and escalate to root — exploitable with a 732-byte Python script, no race conditions required. All major distros shipped since 2017 (Amazon Linux, RHEL, SUSE, Ubuntu) are affected. Critically, the bug also functions as a Kubernetes container escape primitive. Kernel patches landed April 1; CVE publicly disclosed April 29.

## Industry & Trends

### EU Cyber Resilience Act: 2026 Compliance Deadlines Arrive for Software Products
**Source:** European Commission | **URL:** https://digital-strategy.ec.europa.eu/en/policies/cyber-resilience-act

The EU CRA — the world's first horizontal cybersecurity regulation mandating secure-by-design and supply chain security — begins enforcement this year. Type A (horizontal) products must comply by August 30, 2026; Type B/C products by October 30; vulnerability reporting obligations begin September 11. Companies are now legally accountable for the security of open-source components they ship commercially, creating new obligations for any team integrating OSS into products sold in the EU.

### Open Source in 2026: AI Integration, Licensing Battles, and Sustainability Pressure
**Source:** Linux Insider | **URL:** https://www.linuxinsider.com/story/open-source-in-2026-faces-a-defining-moment-177630.html

The 2026 State of Open Source Report shows OSS firmly embedded as core enterprise infrastructure but under converging pressures: the EU CRA imposing new security mandates, AI-adjacent companies shifting to "source available" licensing, and unsustainable funding gaps for critical projects. Vendor lock-in avoidance is now the top driver of OSS adoption, cited by 55% of respondents — a 68% year-over-year increase, reflecting growing cloud and AI dependency concerns.
