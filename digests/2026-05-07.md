### Open-Source LLMs Close the Frontier Gap
_Various Sources_ | [Read more](https://stormap.ai/post/update-on-open-source-ai-model-releases)

Recent releases — Moonshot AI's Kimi-K2.6 (long-context agentic LLM), Qwen3.6-35B-A3B (35B total / 3.5B active parameters via MoE), and Zhipu AI's GLM-5.1 (744B parameter MoE) — mark a decisive shift in the open-source landscape. ### Five Eyes Issue Joint Agentic AI Security Guidance
_CISA / NCSC_ | [Read more](https://cyberscoop.com/cisa-nsa-five-eyes-guidance-secure-deployment-ai-agents/)

Cybersecurity agencies from the US, UK, Canada, Australia, and New Zealand jointly published a 30-page document — "Careful Adoption of Agentic AI Services" — urging organizations to treat autonomous AI agents as a core security concern. The guidance identifies five risk categories: privilege, design/configuration, behavioral, structural (cascading failures across agent networks), and accountability (opaque decision trails). Agencies recommend applying existing frameworks — zero trust, least-privilege, defense-in-depth — rather than treating agentic AI as requiring an entirely new security discipline.

## Security & Privacy

### AI-Discovered CVE-2026-3854 Grants RCE via a Single Git Push on GitHub Enterprise
_Wiz Research / GitHub Blog_ | [Read more](https://github.blog/security/securing-the-git-push-pipeline-responding-to-a-critical-remote-code-execution-vulnerability/)

Wiz Research found CVE-2026-3854 (CVSS 8.7) in GitHub Enterprise Server: unsanitized push option values were injected into internal service headers, allowing any user with repository push access to achieve remote code execution with one `git push`. Notably, this was discovered via AI-assisted reverse engineering of closed-source binaries — a first for a critical vulnerability of this severity. GitHub patched GitHub.com immediately and released an Enterprise Server patch on March 10; no real-world exploitation was detected.

### ShinyHunters Breach Instructure Canvas: 231M Users, 9,000+ Institutions
_TechCrunch / BleepingComputer_ | [Read more](https://techcrunch.com/2026/05/05/hackers-steal-students-data-during-breach-at-education-tech-giant-instructure/)

ShinyHunters breached Instructure — the company behind Canvas LMS, used by 41% of North American higher-ed institutions — claiming 3.65TB stolen and threatening to leak billions of private student/teacher messages unless paid by May 6. Instructure confirmed the April 30 breach exposed names, email addresses, student IDs, and messages across ~231M unique accounts at ~9,000 institutions. Passwords, government IDs, and financial data were reportedly not compromised; Instructure has rotated credentials and engaged outside forensics.

### ADT's Third Breach Since 2024: 5.5M Customers Exposed via Okta SSO Vishing
_BleepingComputer_ | [Read more](https://www.bleepingcomputer.com/news/security/home-security-giant-adt-data-breach-affects-55-million-people/)

ShinyHunters used a voice phishing (vishing) call to impersonate IT support, hijack an ADT employee's Okta SSO session, and pivot into ADT's Salesforce CRM — exfiltrating records on 5.5M customers including names, emails, DOBs, phone numbers, addresses, and partial government IDs. After ADT declined to pay, an 11GB archive was leaked on the dark web. This is ADT's third breach since August 2024, making it a textbook case study in SSO-to-SaaS lateral movement via social engineering.

### DAEMON Tools Supply Chain Attack: Official Installers Backdoored Since April 8
_Kaspersky / BleepingComputer_ | [Read more](https://www.bleepingcomputer.com/news/security/daemon-tools-trojanized-in-supply-chain-attack-to-deploy-backdoor/)

Attackers compromised DAEMON Tools' official website to deliver backdoored installers (versions 12.5.0.2421–12.5.0.2434), signed with valid certificates, that beacon to a C2 domain registered March 27 and download additional payloads. The attack ran undetected from April 8 until Kaspersky disclosed it; ~10% of thousands of infected installs were on organizational systems. A clean version 12.6 was released May 5 — anyone running the compromised versions should treat the host as fully compromised.

### Microsoft SharePoint CVE-2026-32201: 1,300+ Internet-Exposed Servers Still Unpatched
_SecurityWeek / TechRepublic_ | [Read more](https://www.techrepublic.com/article/news-1300-sharepoint-servers-unpatched-zero-day-flaw/)

Despite a patch shipping in April's Patch Tuesday — alongside 168 other CVEs — over 1,300 internet-facing SharePoint servers remain vulnerable to CVE-2026-32201, a spoofing flaw affecting SharePoint Enterprise Server 2016, 2019, and Subscription Edition that was actively exploited as a zero-day. CISA added it to the Known Exploited Vulnerabilities catalog with a federal remediation deadline of April 28. The persistent exposure gap highlights a structural problem with on-premises enterprise patching velocity.

## Cloud & Infrastructure

### NVIDIA Donates GPU DRA Driver to CNCF, Expands Open-Source AI Infrastructure Stack
_NVIDIA Blog / Canonical_ | [Read more](https://canonical.com/blog/canonical-nvidia-kubecon-2026)

At KubeCon Europe 2026, NVIDIA donated its Dynamic Resource Allocation (DRA) Driver for GPUs to the Cloud Native Computing Foundation, transferring governance to the community with backing from AWS, Google Cloud, Microsoft, Red Hat, and Canonical. Simultaneously, NVIDIA's KAI Scheduler entered the CNCF Sandbox, and Grove — an open-source Kubernetes API for declaratively orchestrating AI inference workloads on GPU clusters — was announced. Together, these moves standardize GPU scheduling in Kubernetes and give platform teams vendor-neutral primitives for running large-scale AI workloads.

## Industry & Trends

### CISA Weighs Slashing Patch Deadlines to 3 Days as AI Compresses Attack Windows
_CSO Online / SC Media_ | [Read more](https://www.csoonline.com/article/4167422/cisa-mulls-new-three-day-remediation-deadline-for-critical-flaws.html)

CISA is reportedly considering reducing the remediation deadline for actively exploited vulnerabilities from ~two weeks to three days, driven by evidence that AI-assisted exploitation tools compress the time between disclosure and weaponization from weeks to hours. The proposal is led by CISA acting head Nick Andersen and National Cyber Director Sean Cairncross, though no formal announcement has been made. Critics warn that a 72-hour window may be "simply impossible" for federal agencies running legacy systems that require patch testing, vendor coordination, and operational impact assessment.

_The Hacker News / eSecurity Planet_ | [Read more](https://thehackernews.com/2026/05/2026-year-of-ai-assisted-attacks.html)
