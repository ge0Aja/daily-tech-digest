## Security & Privacy

### Dirty Frag (CVE-2026-43284, CVE-2026-43500): Linux Kernel LPE Hits All Major Distros
_BleepingComputer / Wiz_ | [Read more](https://www.bleepingcomputer.com/news/security/new-linux-dirty-frag-zero-day-with-poc-exploit-gives-root-privileges/)

Dirty Frag chains two page-cache write flaws — xfrm-ESP (CVE-2026-43284) and RxRPC (CVE-2026-43500) — to deliver reliable root access on Ubuntu, RHEL, CentOS Stream, AlmaLinux, openSUSE, and Fedora. The exploit abuses the zero-copy send path where splice() plants a reference to a read-only page cache page (e.g., /etc/passwd) into a sender-side socket buffer, enabling deterministic memory corruption with no race condition required. A public PoC appeared May 7 after an embargo break; the Linux Kernel Organization released patches on May 8. All teams running Linux hosts or containers should apply kernel updates immediately — this is a host-level risk even from containerized environments.

### Copy Fail (CVE-2026-31431): CISA Confirms Active Exploitation of Linux Privilege Escalation
_Microsoft Security Blog_ | [Read more](https://www.microsoft.com/en-us/security/blog/2026/05/01/cve-2026-31431-copy-fail-vulnerability-enables-linux-root-privilege-escalation/)

CVE-2026-31431, dubbed "Copy Fail," is a CVSS 7.8 Linux kernel flaw in the AF_ALG cryptographic socket interface that allows unprivileged users to gain root via a controlled 4-byte kernel page-cache write without modifying files on disk. Affected distributions include Ubuntu 24.04 LTS, Amazon Linux 2023, RHEL 10.1, SUSE 16, Debian, Fedora, and Arch Linux — all kernels since 2017 are vulnerable until patched. Threat actors began exploiting it within 24 hours of researcher disclosure; CISA added it to the KEV catalog with a federal patch deadline of May 15. Because the exploit crosses container boundaries (containers share the host kernel), a compromised container can escalate to host root.

### MongoBleed (CVE-2025-14847): 87K MongoDB Instances Exposed via Heap Memory Leak
_Qualys / InfoQ_ | [Read more](https://www.infoq.com/news/2026/01/mongodb-mongobleed-vulnerability/)

MongoBleed is a CVSS 8.7 critical unauthenticated flaw in MongoDB's zlib network decompression layer that allows attackers to extract credentials, session tokens, and PII directly from server heap memory without authentication. Because zlib compression is enabled by default, 42% of cloud MongoDB deployments were immediately exposed; Censys reports roughly 87,000 internet-facing instances remain unpatched. Affected versions span MongoDB 3.6 through 8.2.2; patched versions are 8.2.3, 8.0.17, 7.0.28, 6.0.27, and 5.0.32. Teams should upgrade and rotate all credentials accessible from or stored in affected instances.

### Windows NTLM Zero-Click Hash Leak Under Active Exploitation (CVE-2026-32202)
_BleepingComputer / Akamai_ | [Read more](https://www.bleepingcomputer.com/news/security/cisa-orders-feds-to-patch-windows-flaw-exploited-in-zero-day-attacks/)

CVE-2026-32202 is a zero-click NTLM hash leak in Windows — the result of an incomplete February 2026 patch for RCE flaw CVE-2026-21510 — that allows attackers to capture NTLM credentials without user interaction simply by having the target view a malicious file or directory. Discovered by Akamai researchers, it is actively exploited in the wild; CISA's federal patch deadline is May 12. All Windows environments using NTLM authentication (most Active Directory deployments) should apply the patch immediately or disable NTLM as a compensating control where feasible.

### Redis 8 Adopts Tri-License: RSAL 2.0, SSPL v1, and AGPLv3
_OpenLogic_ | [Read more](https://www.openlogic.com/blog/top-open-source-databases)

Redis 8 introduced a tri-licensing model in early 2026, available under the Redis Source Available License 2.0, the Server Side Public License v1, and the AGPLv3. The AGPLv3 option restores a path to OSI-compliant usage after the controversial 2024 license change that dropped Redis from OSI's approved list and triggered the creation of the Valkey fork. SSPL continues to restrict cloud provider redistribution; the AGPLv3 path is available for teams that can accept copyleft obligations. Teams using Redis in production or as a dependency should audit their licensing requirements, particularly for SaaS products where AGPL's network-use copyleft clause triggers.
