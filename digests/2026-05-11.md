### Google Cloud Next '26: 8th-Gen TPUs and Planetary-Scale AI Infrastructure
_Google Cloud Blog_ | [Read more](https://cloud.google.com/blog/products/compute/ai-infrastructure-at-next26)

Google unveiled its 8th-generation Tensor Processing Units at Cloud Next '26, introducing two specialized chips: TPU 8t for training (up to 9,600 chips per superpod delivering 121 exaflops and 2 PB of shared HBM, ~3× the compute of prior generations) and TPU 8i for inference (3× on-chip SRAM, 5× lower collective latency, 80% better performance/dollar). These announcements reset the baseline for large-scale AI training and inference on public cloud and signal that hyperscaler infrastructure gaps between cloud and on-prem are closing fast.

### Cloudflare Builds High-Performance LLM Inference Infrastructure with Unweight Compression
_Cloudflare Blog_ | [Read more](https://blog.cloudflare.com/high-performance-llms/)

Cloudflare published the engineering details of its LLM inference optimizations: disaggregated prefill separates input processing from token generation across specialized hardware; speculative decoding uses a smaller draft model to pre-generate candidate tokens for the larger model to verify; and prompt caching improvements boosted cache hit ratios from 60% to 80% at peak load, significantly increasing throughput for interactive sessions. The company also developed "Unweight," a lossless inference-time compression system achieving up to 22% reduction in model footprint for faster, cheaper inference without quality loss. Combined with a Workers AI integration supporting 14+ model providers from a single API surface, Cloudflare is positioning its edge network as a production-grade inference tier that competes with dedicated cloud AI services.

### Cloudflare Project Think: A Durable, Batteries-Included Runtime for Agentic AI
_Cloudflare Blog_ | [Read more](https://blog.cloudflare.com/project-think/)

Launched April 30, 2026, Project Think ships a full agent platform on Cloudflare Workers: durable fibers for crash recovery and checkpointing, sub-agents with isolated SQLite databases, persistent tree-structured sessions, and Dynamic Workers — isolate-based sandboxes that spin up in milliseconds to safely execute LLM-generated code without container overhead. Agents can self-author new tools at runtime, spawn sub-agents, and autonomously provision Cloudflare infrastructure (accounts, domains, API tokens, deployments). The full stack offers zero idle cost via hibernation, per-agent isolation via Durable Objects, sandboxed OS access, and LLM connectivity through Workers AI. For teams building multi-step agentic workflows, this is a compelling serverless substrate with global distribution, strong isolation, and no infrastructure management.

## Security & Privacy

### Critical RCE in Microsoft Semantic Kernel: Prompt Injection Becomes Shell Access
_Microsoft Security Blog_ | [Read more](https://www.microsoft.com/en-us/security/blog/2026/05/07/prompts-become-shells-rce-vulnerabilities-ai-agent-frameworks/)

Microsoft disclosed two critical remote code execution vulnerabilities in the Semantic Kernel framework: CVE-2026-26030 (Python SDK <1.39.4) exploits an `eval()` call in the In-Memory Vector Store's default string filter, turning a single crafted prompt into host-level RCE; CVE-2026-25592 (.NET SDK <1.71.0) accidentally exposes `DownloadFileAsync` from `SessionsPythonPlugin` to the model, enabling arbitrary file writes to host startup directories that bypass container isolation. Both vulnerabilities require no memory corruption or browser exploits — prompt injection alone is the complete attack vector. Teams using Semantic Kernel must upgrade immediately to Python ≥1.39.4 or .NET ≥1.71.0 and audit existing deployments for signs of exploitation.

### ShinyHunters Breach Canvas LMS: 275M Records From 8,800+ Schools — Largest Education Breach Ever
_CNN / Wikipedia_ | [Read more](https://en.wikipedia.org/wiki/2026_Canvas_security_incident)

On April 25, 2026, ShinyHunters compromised Instructure's Canvas LMS, exfiltrating an estimated 3.65 TB of data — roughly 275 million records spanning 8,809 educational institutions across the US, UK, Canada, Australia, Singapore, and more. Stolen data includes student and staff names, email addresses, student IDs, and what ShinyHunters claims are "several billion private messages." The group issued a May 12 ransom deadline, disrupted platform access during US finals week, and affected institutions including Harvard, multiple San Diego-area campuses, and thousands of other schools worldwide. Canvas serves 41% of North American higher education — making this the largest educational data breach on record and a critical reminder that LMS platforms hold sensitive PII at extraordinary scale.

### 1 Million Exposed AI Services: LLM Self-Hosted Infrastructure Is Dangerously Misconfigured
_The Hacker News_ | [Read more](https://thehackernews.com/2026/05/we-scanned-1-million-exposed-ai.html)

A security research scan of 2M+ hosts found that self-hosted LLM infrastructure is among the most insecure software ever analyzed at scale: many projects ship with authentication disabled by default, real user data and internal tooling are exposed to the public internet, and processes commonly run as root. Recurring failure modes include hardcoded credentials, misconfigured Docker setups, and unmodified default installs deployed straight into production. The root cause is organizational pressure — teams are moving fast to self-host AI infrastructure and bypassing security hardening in the process, undoing two decades of hard-won industry progress on secure defaults. Engineers deploying LLM frameworks must treat authentication enforcement, network segmentation, and least-privilege process isolation as table stakes, not optional hardening.

_InfoQ_ | [Read more](https://www.infoq.com/news/2026/05/gke-agent-sandbox-hypercluster/)

Google announced two major GKE capabilities at Cloud Next '26. For platform teams managing AI workloads at scale, these features meaningfully reduce both the security blast radius of agentic code execution and the cluster sprawl that currently plagues large ML infrastructure.

## Industry & Trends

### EU Cyber Resilience Act: Mandatory Vulnerability Reporting Begins September 11, 2026
_HeroDevs_ | [Read more](https://www.herodevs.com/blog-posts/cra-reporting-obligations-start-september-2026-what-eol-dependencies-mean-for-your-compliance)

Starting September 11, 2026, the EU Cyber Resilience Act requires manufacturers selling software products in Europe to report actively exploited vulnerabilities to ENISA and national CSIRTs within 24 hours of discovery — and to notify affected users without undue delay. The European Commission's March 2026 draft guidance clarifies that volunteer-driven open source projects are largely exempt, but commercial entities distributing or embedding OSS are accountable for the security of those components, including EOL dependencies. With the deadline less than four months away, teams shipping EU-facing products should begin open-source dependency audits immediately and establish internal vulnerability triage and disclosure workflows.
