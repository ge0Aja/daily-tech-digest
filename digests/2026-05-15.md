## Security & Privacy

### Mini Shai-Hulud Worm Compromises TanStack, Mistral AI, and 160+ npm/PyPI Packages
_Wiz Blog_ | [Read more](https://www.wiz.io/blog/mini-shai-hulud-strikes-again-tanstack-more-npm-packages-compromised)

On May 11, the TeamPCP group published 84 malicious versions across 42 @tanstack/* npm packages using a novel attack chain: GitHub Actions cache poisoning across the fork/base trust boundary combined with OIDC token extraction from CI runner process memory — no npm credentials stolen. The self-propagating worm spread to Mistral AI, UiPath, Guardrails AI, and 160+ additional packages; @tanstack/react-router alone has 12.7M weekly downloads. Malware exfiltrates credentials over the Oxen messenger network and installs persistent polling daemons. Anyone who installed affected versions must immediately rotate all cloud (AWS, GCP, Kubernetes, Vault), GitHub, npm, and SSH credentials.

### OpenAI Confirms Data Breach Linked to TanStack Supply Chain Attack
_BleepingComputer_ | [Read more](https://www.bleepingcomputer.com/news/security/openai-confirms-security-breach-in-tanstack-supply-chain-attack/)

Two OpenAI employee devices were compromised in the Mini Shai-Hulud campaign, resulting in unauthorized access to a subset of internal source code repositories that contained code-signing certificates. OpenAI confirmed no user data, production systems, or intellectual property was accessed, but is rotating certificates as a precaution — requiring macOS app users to update. This breach illustrates a critical lesson: even security-focused organizations with strict controls are exposed when their developers' machines install compromised open-source dependencies.

May 2026 brings two critical network infrastructure vulnerabilities: CVE-2026-20182 in Cisco Catalyst SD-WAN Controller and Manager allows unauthenticated remote attackers to bypass authentication entirely and gain full administrative access (CVSS 10.0), while NGINX Rift (CVE-2026-42945, CVSS 9.2) is a heap buffer overflow enabling remote code execution or denial of service. _Databricks Blog_ | [Read more](https://www.databricks.com/blog/next-era-open-lakehouse-apache-icebergtm-v3-public-preview-databricks)

Databricks released Apache Iceberg v3 in public preview, introducing Deletion Vectors (eliminating full-file rewrites on updates and deletes), Row Lineage, and VARIANT data type support — delivering up to 10x faster data manipulation vs. traditional copy-on-write approaches.
