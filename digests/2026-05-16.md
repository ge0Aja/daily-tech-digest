## Security & Privacy

### Mini Shai-Hulud Worm Poisons 400+ npm/PyPI Packages; OpenAI Confirms Exposure
_HackRead / Wiz Blog / BleepingComputer_ | [Read more](https://hackread.com/teampcp-mini-shai-hulud-worm-npm-pypi-packages/)

On May 11, the TeamPCP extortion group launched "Mini Shai-Hulud," a supply chain worm that compromised over 400 npm and PyPI packages — including TanStack, Mistral AI, Guardrails AI, UiPath, and OpenSearch — within five hours by hijacking OIDC tokens from legitimate CI/CD pipelines without stealing any credentials. The malware exfiltrates AWS IAM credentials, HashiCorp Vault secrets, and GitHub tokens, then moves laterally through victim infrastructure. OpenAI confirmed two employee devices were compromised and that code-signing certificates used to validate OpenAI software releases were among the exfiltrated material; any developer who installed affected packages should immediately rotate all cloud, GitHub, npm, and SSH credentials.

### Cisco SD-WAN CVE-2026-20182: CVSS 10.0 Auth Bypass Actively Exploited, CISA Deadline May 17
_Help Net Security / The Hacker News_ | [Read more](https://www.helpnetsecurity.com/2026/05/15/cisco-sd-wan-zero-day-cve-2026-20182/)

A critical authentication bypass in Cisco Catalyst SD-WAN Controller (CVE-2026-20182, CVSS 10.0) allows unauthenticated remote attackers to gain full administrative access by sending crafted DTLS requests that exploit broken peer certificate verification in the vdaemon service. Active exploitation is attributed with high confidence to UAT-8616, the same threat cluster behind a prior SD-WAN zero-day. CISA added it to the Known Exploited Vulnerabilities catalog with a federal remediation deadline of May 17, 2026 — no workaround exists, patching is the only fix.

### Microsoft Exchange CVE-2026-42897: XSS Spoofing Bug Weaponized via Crafted Email
_The Hacker News_ | [Read more](https://thehackernews.com/2026/05/on-prem-microsoft-exchange-server-cve.html)

CVE-2026-42897 is a spoofing vulnerability (CVSS 8.1) in on-premises Microsoft Exchange Server caused by a cross-site scripting flaw in Outlook Web Access. Attackers exploit it by sending specially crafted emails; when the recipient opens the message in OWA, arbitrary JavaScript executes in their browser session — a low-friction attack vector requiring no special privileges beyond the ability to send email. Organizations still running on-prem Exchange should apply the patch immediately, as the attack chain is straightforward and requires minimal user interaction.

### Open Source in 2026: AI Licensing Battles Threaten to Redefine "Open"
_Linux Insider / Open Source Initiative_ | [Read more](https://www.linuxinsider.com/story/open-source-in-2026-faces-a-defining-moment-177630.html)

The OSI's Open Source AI Definition (OSAID 1.0) requires published training data information, complete training code, and unrestricted model weights — but by that strict standard, almost no popular "open" AI model qualifies. Tensions are escalating: MiniMax recently shifted its M2.7 model from MIT to a non-commercial license, while 55% of enterprises now cite vendor lock-in avoidance as their top open-source adoption driver (up 68% YoY). Engineering teams should audit the licensing of AI model dependencies now, as the legal and contractual weight of "open" AI is becoming increasingly significant for production deployments.
