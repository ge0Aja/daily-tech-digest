**Source:** BleepingComputer | [Read more](https://www.bleepingcomputer.com/news/security/github-links-repo-breach-to-tanstack-npm-supply-chain-attack/)

Any project using affected `@tanstack/*` packages should rotate CI/CD secrets and audit pipeline permissions immediately.

### NGINX CVE-2026-42945: 18-Year-Old Heap Overflow Actively Exploited in the Wild
**Source:** The Hacker News | [Read more](https://thehackernews.com/2026/05/nginx-cve-2026-42945-exploited-in-wild.html)

CVE-2026-42945 (CVSS 9.2, dubbed "NGINX Rift") is a heap buffer overflow in NGINX's `ngx_http_rewrite_module`, present in NGINX Open Source 0.6.27–1.30.0 and NGINX Plus vR32–R36 — a flaw introduced in 2008. A crafted HTTP request can reliably trigger a DoS condition; on systems with ASLR disabled it can enable unauthenticated remote code execution. A public PoC appeared May 13 and active exploitation was detected by honeypot networks by May 16. Patching or applying F5's mitigations is urgent, especially for any NGINX instance without ASLR enforced.

### Discord Enables End-to-End Encryption by Default for All Voice and Video Calls
**Source:** BleepingComputer | [Read more](https://www.bleepingcomputer.com/news/security/discord-rolls-out-end-to-end-encryption-on-voice-video-calls/)

Discord has rolled out end-to-end encryption by default for all voice and video calls — covering DMs, group DMs, voice channels, and Go Live streams — across desktop, mobile, web, and gaming consoles. The open-source DAVE protocol, introduced in 2024, is now the only code path; the unencrypted fallback has been removed entirely. Stage channels are exempt due to their broadcast nature, and text messages remain outside the E2EE scope, but this is a significant privacy improvement for Discord's 500M+ registered users.

### etcd 3.7.0-beta.0 Released — RangeStream and Full v3store Migration
**Source:** Kubernetes Blog | [Read more](https://kubernetes.io/blog/2026/05/20/etcd-370-beta/)

SIG-Etcd released the first beta of etcd v3.7.0, featuring the long-awaited RangeStream RPC that lets clients receive large key-value result sets in chunks rather than blocking on the complete response.
