**Source:** The Hacker News | [Read more](https://thehackernews.com/2026/05/laravel-lang-php-packages-compromised.html)

Attackers compromised four popular Laravel localization packages (laravel-lang/lang, http-statuses, attributes, actions) on May 22–23 by publishing 700+ malicious version tags pointing to a controlled fork — never committing to the official repos, bypassing standard commit-history auditing. The payload (~5,900 lines, 15 specialist collector modules) scrapes environment variables, .env files, database credentials, API keys, and CI secrets; encrypts with AES-256; exfiltrates to `flipboxstudio[.]info/exfil`; then self-deletes. Any project that resolved these packages via Composer during the May 22–23 window should rotate all CI secrets, API keys, and database credentials immediately and review job logs for unexpected outbound connections.

### Megalodon: Automated Campaign Backdoors 5,561 GitHub Repos via CI/CD Workflow Poisoning in Six Hours
**Source:** The Hacker News | [Read more](https://thehackernews.com/2026/05/megalodon-github-attack-targets-5561.html)

On May 18, an automated campaign pushed 5,718 malicious commits to 5,561 public GitHub repos in six hours using throwaway accounts with forged CI bot identities (build-bot, auto-ci, pipeline-bot). Two payload variants: SysDiag triggers on every push/PR for immediate execution; Optimize-Build creates dormant backdoors activated on demand via the GitHub API. Both exfiltrate AWS keys, GCP tokens, Azure IMDS credentials, SSH private keys, Docker/Kubernetes configs, Vault tokens, Terraform credentials, and 30+ secret regex patterns to a C2 at 216.126.225.129:8443. Maintainers should audit `.github/workflows/` for unknown files, cross-reference commit authors against expected contributors, and specifically review changes from the May 18 window.
