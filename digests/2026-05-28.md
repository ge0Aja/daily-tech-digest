### CVE-2026-42897: Microsoft Exchange OWA XSS Actively Exploited — Patch by May 29
**Source:** The Hacker News / CISA | [Read more](https://thehackernews.com/2026/05/on-prem-microsoft-exchange-server-cve.html)

A high-severity (CVSS 8.1) stored XSS in Microsoft Exchange's Outlook Web Access is being actively exploited: crafted emails, when opened in OWA, execute arbitrary JavaScript in the victim's browser. Exchange Server 2016, 2019, and SE (all currently supported on-prem versions) are affected. CISA added it to the KEV catalog on May 15 with a federal remediation deadline of May 29; organizations should treat this as urgent given OWA's exposure in hybrid environments. The Exchange Emergency Mitigation Service (EEMS) provides automatic interim mitigation for deployments that cannot patch immediately.

### Cisco SD-WAN CVE-2026-20182: Critical Auth Bypass — Sixth SD-WAN Zero-Day Exploited in 2026
**Source:** SecurityWeek / Cisco Talos | [Read more](https://www.securityweek.com/cisco-patches-another-sd-wan-zero-day-the-sixth-exploited-in-2026/)

CVE-2026-20182 is a critical (CVSS 10.0) authentication bypass in Cisco Catalyst SD-WAN Controller, actively exploited by threat actor UAT-8616 since at least May 2026. The flaw targets the vdaemon DTLS service (UDP 12346): peers claiming to be vHub devices skip certificate verification yet are still marked as authenticated, granting remote admin access. Post-compromise activity includes SSH key injection, NETCONF configuration manipulation, malicious account creation, and extensive log clearing. Patches are available for all supported SD-WAN releases; CISA issued Emergency Directive 26-03 with a May 17 remediation deadline.

### Gitea CVE-2026-27771: Private Container Images Exposed Without Auth for ~4 Years Across 30,000+ Deployments
**Source:** The Hacker News / TechTimes | [Read more](https://thehackernews.com/2026/05/gitea-vulnerability-exposes-private.html)

A CVSS 8.2 flaw in all Gitea versions prior to 1.26.2 allowed unauthenticated remote access to pull private container images without credentials of any kind. The bug went undetected for nearly four years, potentially exposing 30,000+ deployments in 30+ countries spanning healthcare, aerospace, retail, and ISPs. The "private" designation on container repositories simply did not enforce the access controls operators reasonably expected. Update to Gitea 1.26.2 immediately; if patching is not possible, set `[service].REQUIRE_SIGNIN_VIEW=true` in your Gitea config as a short-term workaround.

**Source:** Cloud Native Now | [Read more](https://cloudnativenow.com/contributed-content/why-longer-kubernetes-release-cycles-are-critical-for-private-cloud-adoption/)

As sovereign and private cloud deployments surge — Gartner projects $80.4B in sovereign cloud spending in 2026, a 35% year-on-year increase — the Kubernetes community is revisiting its rapid quarterly release cadence. Private cloud operators and regulated industries consistently run multiple versions behind because they cannot absorb frequent upgrades at the pace public cloud providers can.
