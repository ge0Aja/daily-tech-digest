## AI & Machine Learning

### MiniMax M3: Open-Weight Frontier Model with 1M Token Context Challenges Proprietary Leaders
**Source:** The Decoder | [Read more](https://the-decoder.com/minimax-m3-open-weight-model-with-a-million-token-context-challenges-proprietary-leaders/)

Released June 1 by Shanghai-based MiniMax, M3 is the first open-weight model to combine frontier coding performance, a one-million-token context window, and native multimodality in a single architecture. The key innovation is MiniMax Sparse Attention (MSA), which slashes per-token compute to 1/20th of prior models at full context, delivering 9x faster prefilling and 15x faster decoding. On SWE-Bench Pro M3 scores 59% — ahead of GPT-5.5 and Gemini 3.1 Pro — and model weights are scheduled for public release within 10 days of launch.

### When Prompts Become Shells: RCE Vulnerabilities in Microsoft Semantic Kernel (CVE-2026-25592, CVE-2026-26030)
**Source:** Microsoft Security Blog | [Read more](https://www.microsoft.com/en-us/security/blog/2026/05/07/prompts-become-shells-rce-vulnerabilities-ai-agent-frameworks/)

Microsoft disclosed two critical vulnerabilities in Semantic Kernel — CVE-2026-25592 (.NET SDK) and CVE-2026-26030 (Python SDK) — where prompt injection escalates to host-level remote code execution with no traditional exploit required. Attackers craft malicious prompts that bypass AST validation via Python class hierarchy traversal, dynamically load the `os` module, and execute arbitrary system commands. Engineers must upgrade the .NET SDK to ≥1.71.0 and the Python SDK to ≥1.39.4 immediately; any agent framework registering dangerous tool primitives (eval, exec, filesystem writes) is vulnerable to this class of attack.

## Security & Privacy

### Cisco SD-WAN CVE-2026-20245: Seventh Zero-Day of 2026 Actively Exploited, No Patch Available
**Source:** Help Net Security | [Read more](https://www.helpnetsecurity.com/2026/06/05/cisco-sd-wan-cve-2026-20245-0-day-exploited/)

Cisco disclosed an unpatched high-severity zero-day (CVE-2026-20245, CVSS 7.8) in Catalyst SD-WAN Manager allowing authenticated attackers with netadmin privileges to escalate to root by uploading a crafted file via the CLI. The flaw — the seventh exploited SD-WAN zero-day in 2026 — was reported by Mandiant after observing active exploitation that included attacker-pushed configuration changes to downstream edge devices. No patch or workaround is currently available; Cisco advises upgrading to the release that addressed the earlier CVE-2026-20182 as partial mitigation.

### IronWorm: Rust-Written Infostealer Hits 36 npm Packages in Supply-Chain Attack
**Source:** BleepingComputer | [Read more](https://www.bleepingcomputer.com/news/security/new-ironworm-malware-hits-36-packages-in-npm-supply-chain-attack/)

A Rust-based infostealer called IronWorm compromised 36 npm packages starting from a hijacked account, targeting 86 environment variables and 20 credential files including OpenAI, AWS, Anthropic, npm, SSH keys, and Exodus cryptocurrency wallet data. The malware hides behind an eBPF kernel rootkit, communicates via Tor, and self-propagates using stolen npm Trusted Publishing credentials to infect downstream developers and CI systems. Developers should audit dependencies against the published affected package list, rotate all secrets, and enable 2FA on all npm accounts.

### DentaQuest Data Breach: ShinyHunters Dumps 234 GB, Exposing 2.6 Million Accounts
**Source:** BleepingComputer | [Read more](https://www.bleepingcomputer.com/news/security/dentaquest-data-breach-exposed-info-of-26-million-accounts/)

Sun Life subsidiary DentaQuest confirmed on June 2 that its network was breached after the ShinyHunters extortion group — having failed to negotiate a ransom — published 234 GB of stolen data affecting 2.6 million people. Exposed records include names, emails, phone numbers, government-issued IDs, Medicaid IDs, and health insurance data extracted from ASC X12 healthcare enrollment files. DentaQuest serves approximately 35 million US customers across Medicaid, Medicare Advantage, and employer benefit plans.

## Systems & Engineering

### Kubernetes Introduces Agent Sandbox: Native Isolation for AI Agent Workloads
**Source:** Kubernetes Blog | [Read more](https://kubernetes.io/blog/2026/03/20/running-agents-on-kubernetes-with-agent-sandbox/)

The Kubernetes project introduced Agent Sandbox, a new custom resource providing strong kernel and network isolation for AI agent workloads that execute untrusted or non-deterministic code. The sandbox natively supports gVisor and Kata Containers runtimes, giving cluster operators configurable isolation tiers without patching each workload individually. With 66% of organizations now running generative AI on Kubernetes and multi-agent workflow counts up 327% in recent months, this fills a critical gap in the platform's security model for multi-tenant AI deployments.

### Securing Autonomous AI Agents on Kubernetes: Trust Boundaries, Secrets, and Observability
**Source:** InfoQ | [Read more](https://www.infoq.com/articles/securing-autonomous-ai-agents-kubernetes/)

A deep-dive from InfoQ argues that autonomous AI agents are a fundamentally new category of cloud workload that breaks standard Kubernetes RBAC and pod security assumptions — they run long-lived processes with broad tool access, generate non-deterministic behavior, and chain calls across external services. The article outlines patterns for trust boundary segmentation between agent components, secrets management strategies that avoid over-privileging persistent agent processes, and observability pipelines for detecting anomalous agent behavior. Treating agent pods as first-class workloads with dedicated threat models is now a practical engineering requirement.

## Industry & Trends

### EU AI Act Overhauled: Digital Omnibus Extends High-Risk Deadlines Up to 16 Months
**Source:** Inside Privacy | [Read more](https://www.insideprivacy.com/artificial-intelligence/eu-ai-act-update-timeline-relief-targeted-simplification-and-new-prohibitions/)

On May 7, EU negotiators agreed to the Digital Omnibus — the first amendment to the AI Act since 2024 — deferring Annex III high-risk AI system obligations by 16 months to December 2027 and expanding SME thresholds to companies with up to 750 employees or €150M revenue. Two new prohibitions take effect December 2026: AI-generated child sexual abuse material and non-consensual synthetic intimate imagery. Formal adoption is expected in June with publication in July, giving companies deploying high-risk AI systems in the EU a significant compliance extension.
