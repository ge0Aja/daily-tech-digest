### LangGraph Critical RCE Vulnerability Chain Exposes 46M-Download AI Agent Framework
**Source:** The Hacker News | [Read more](https://thehackernews.com/2026/06/langgraph-flaw-chain-exposes-self.html)

Researchers disclosed a three-CVE vulnerability chain in LangGraph — an AI agent framework with 46.5 million monthly downloads — affecting self-hosted deployments using SQLite or Redis checkpointers. The chain involves CVE-2025-67644 (SQLite injection in `get_state_history()`), CVE-2026-28277 (msgpack deserialization RCE), and CVE-2026-27022 (Redis injection); chaining the first two achieves full remote code execution, exposing LLM API keys, customer data, and credentials connected to external systems. Patches: upgrade to `langgraph-checkpoint-sqlite ≥ 3.0.1`, `langgraph ≥ 1.0.10`, and `langgraph-checkpoint-redis ≥ 1.0.2`; managed LangSmith deployments are not affected.

### 2026 OSSRA Report: AI Code Generation Doubles Open-Source Vulnerabilities, License Conflicts Hit 68%
**Source:** Black Duck | [Read more](https://www.blackduck.com/blog/open-source-trends-ossra-report.html)

Black Duck's 2026 Open Source Security and Risk Analysis (OSSRA) report, based on audits of 947 commercial codebases across 17 industries, found open-source vulnerabilities doubled year-over-year while license conflicts reached 68% of audited codebases — up from 56%, the largest single-year increase in the report's history. The primary driver of the licensing surge is "license laundering": AI code assistants generating snippets derived from copyleft (GPL/LGPL) sources without retaining original license metadata, unknowingly introducing compliance obligations at scale. The report concludes that AI is simultaneously amplifying risk across security, licensing, and operational sustainability, making automated dependency governance a high-priority engineering concern.
