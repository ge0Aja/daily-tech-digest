**Source:** OpenAI | [Read more](https://openai.com/index/deployment-simulation/)

### ServiceNow Unauthenticated API Flaw Exploited Against Enterprise Customers (No CVE Assigned)
**Source:** Bleeping Computer | [Read more](https://www.bleepingcomputer.com/news/security/servicenow-discloses-security-incident-exposing-customer-data/)

ServiceNow disclosed on June 9 that attackers exploited an unauthenticated access flaw in a Scripted REST API endpoint — `/api/now/related_list_edit/create` — configured with `requires_authentication=false`, allowing unauthenticated HTTP requests to query enterprise customer instance data without valid credentials. The exploitation window was June 2–3; ServiceNow had received a bug bounty report on April 22 but did not apply a fix until June 5, after active exploitation began. The incident primarily affected customers on the Australia platform release or with specific configurations on older releases; no CVE has been assigned. Engineers maintaining ServiceNow environments or building on its REST APIs should audit all Scripted REST API endpoints for authentication configuration and review access logs for the June 2–3 window.
