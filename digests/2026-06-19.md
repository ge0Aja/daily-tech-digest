### Google Vertex AI SDK "Pickle in the Middle": Predictable Bucket Names Enabled Model Hijacking
**Source:** The Hacker News | [Read more](https://thehackernews.com/2026/06/google-vertex-ai-sdk-flaw-let-attackers.html)

Palo Alto Networks Unit 42 disclosed a flaw — dubbed "Pickle in the Middle" — in the Google Cloud Vertex AI SDK for Python that allowed an attacker to hijack a victim's ML model uploads and execute code inside Google's serving infrastructure without any access to the victim's project. The attack exploited the SDK's use of predictable default Cloud Storage bucket names (derived from project ID and region); since project IDs are often public, the attacker could pre-register the bucket and intercept uploads. Google patched the issue in SDK v1.144.0 (March 31) and completed the fix in v1.148.0 (April 15); teams still on older SDK versions should upgrade immediately and explicitly define staging buckets rather than relying on SDK defaults.
