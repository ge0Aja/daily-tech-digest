### CISA Deadline TODAY: Splunk Enterprise CVE-2026-20253 (CVSS 9.8) — Unauthenticated RCE Under Active Exploitation
**Source:** Help Net Security / SOCRadar | [Read more](https://www.helpnetsecurity.com/2026/06/19/splunk-vulnerability-cve-2026-20253-exploited/)

CVE-2026-20253 is a critical (CVSS 9.8) pre-authentication remote code execution vulnerability in Splunk Enterprise versions 10.0.0–10.0.6 and 10.2.0–10.2.3, rooted in an unauthenticated PostgreSQL sidecar service endpoint that allows arbitrary file operations from any network-reachable attacker. The exploitation chain involves writing a malicious database dump via the `/backup` endpoint, restoring it via `/restore`, then leveraging the PostgreSQL sidecar to execute arbitrary code — achieving full Splunk environment compromise with access to stored credentials, security logs, and lateral movement paths. CISA added it to the Known Exploited Vulnerabilities catalog on June 18 — the **first-ever Splunk entry in the KEV catalog** — with a US federal remediation deadline of **today, June 21**. Patch to Splunk Enterprise 10.0.7 or 10.2.4 immediately; if patching is not possible, network-isolate Splunk instances from general network access.

**Source:** The Hacker News / Help Net Security | [Read more](https://thehackernews.com/2026/06/chrome-v8-zero-day-cve-2026-11645.html)

Update Chrome to 149.0.7827.102/.103 for Windows/macOS or 149.0.7827.102 for Linux; teams managing Chromium-based apps or Electron applications should verify they're running the patched V8 build.

Apple released Beats Firmware Update 1B211 on June 16 to fix CVE-2025-20701 (CVSS 8.8), a vulnerability in the Airoha Bluetooth audio SoC SDK that allows an attacker within Bluetooth range to pair with an unpaired device without user consent and eavesdrop through its microphone. The firmware patch is delivered automatically when earbuds are within Bluetooth range of a paired iPhone, iPad, or Mac; this is a supply-chain SoC issue with broad implications for Bluetooth audio hardware used in enterprise environments, open offices, and on-call rotations.

### EU Cyber Resilience Act Vulnerability Reporting Kicks In September 2026 — Three Months to Comply
**Source:** OpenSSF / BCLP | [Read more](https://openssf.org/category/policy/cra/)

The EU Cyber Resilience Act's vulnerability reporting obligations take effect in September 2026, three months away, requiring manufacturers of products with digital elements to report actively exploited vulnerabilities to ENISA within 24 hours of discovery and publish fixes within 14 days. The broader compliance requirements (Articles 9–17: risk management, security testing, SBOM, update mechanisms, and human oversight) apply from December 2027, but the September reporting obligations are binding and carry penalties up to €15M or 2.5% of global turnover. Open source maintainers whose software is distributed commercially or monetized fall in scope; personal non-commercial contributors generally do not. Organizations shipping software into the EU should immediately audit vulnerability disclosure workflows, SBOM generation pipelines, and incident response SLAs against the 24-hour/14-day CRA reporting windows.
