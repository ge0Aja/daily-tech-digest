### FortiBleed: 75,000 Fortinet Firewall Credentials Cracked with 45-GPU Cluster — Patched Devices Also Affected
**Source:** Computing / SOCRadar | [Read more](https://www.computing.co.uk/news/2026/security/fortibleed-leaked-admin-credentials-expose-nearly-74-000-fortinet-firewalls-worldwide)

Security researcher Volodymyr "Bob" Diachenko discovered on June 17 an attacker-controlled server holding over 86,644 verified working admin credentials for Fortinet FortiGate firewalls and SSL VPN devices spanning 194 countries — a campaign now dubbed FortiBleed. The attackers swept 59.3 million internet-facing hosts, executed 1.16 billion credential-based login attempts against 320,777 FortiGate targets, and cracked intercepted SSL VPN authentication hashes offline using a 45-GPU Hashtopolis cluster — critically, this succeeded even on devices that had been patched, since the hashes were captured from live sessions before patching. The scale and method — offline GPU cracking of intercepted hashes rather than exploitation of an unpatched flaw — means standard patch compliance workflows provide an incomplete defense. Organizations running Fortinet infrastructure should immediately rotate all admin and SSL VPN credentials, audit active sessions for anomalous access, and review VPN connection logs from the past 90 days for signs of unauthorized entry.

### F5 Patches Critical NGINX HTTP/3 RCE (CVE-2026-42530, CVSS 9.2) — Partial Fix, Ingress Controller Still Unpatched
**Source:** The Hacker News / SecurityWeek | [Read more](https://thehackernews.com/2026/06/f5-patches-two-critical-nginx-open.html)

On June 17, F5 issued out-of-band patches for two critical NGINX vulnerabilities: CVE-2026-42530 (CVSS 9.2) and CVE-2026-42055, both in the HTTP/3 QUIC module. CVE-2026-42530 is a Use-After-Free in the ngx_http_v3_module: a remote unauthenticated attacker can send a specially crafted HTTP/3 session to reopen a QPACK encoder stream, causing a worker-process restart and enabling arbitrary code execution on systems with ASLR disabled. Affected products include NGINX Open Source 1.31.0–1.31.1, NGINX Gateway Fabric 1.3.0–2.6.3, NGINX Instance Manager 2.17.0–2.22.0, and NGINX Ingress Controller 3.x/4.x/5.x; patches are currently available only for NGINX Open Source (1.31.2) and Gateway Fabric (2.6.4) — the Ingress Controller and Instance Manager remain unpatched. Organizations that cannot immediately upgrade should disable HTTP/3 by removing `quic` from all `listen` directives; no active exploitation has been confirmed as of the patch release date.

**Source:** Bleeping Computer / JetBrains Blog | [Read more](https://www.bleepingcomputer.com/news/security/malicious-jetbrains-marketplace-plugins-steal-ai-api-keys-from-developers/)

The plugins impersonated legitimate AI coding assistants, code-review tools, and Git utilities supporting OpenAI, DeepSeek, and SiliconFlow — and fired a key-exfiltration call the moment a user clicked Apply in the settings dialog, with no visible indication of outbound network activity. JetBrains removed the plugins from Marketplace, blocked the publisher accounts, and pushed a remote disable to already-installed copies; any developer who configured an AI API key in a JetBrains IDE in the past eight months should rotate OpenAI, DeepSeek, or SiliconFlow credentials immediately.
