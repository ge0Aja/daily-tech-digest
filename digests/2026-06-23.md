### FortiBleed: 73,932 Fortinet Firewall Admin Credentials Leaked — No CVE, No Patch
**Source:** Help Net Security / CISA | [Read more](https://www.helpnetsecurity.com/2026/06/18/fortinet-fortibleed-data-leak/)

A credential leak campaign dubbed "FortiBleed" has exposed verified administrator credentials for approximately 73,932 internet-facing FortiGate firewalls across 194 countries as of June 18, potentially affecting 50% of all reachable FortiGate devices globally. Attackers used automated scanning combined with offline GPU-accelerated cracking of intercepted SSL VPN authentication hashes via Hashtopolis clusters — critically, this succeeded even against devices that had already been patched, since hashes were captured from live sessions before patching occurred. There is no CVE and no vendor patch to apply; Fortinet states the dataset aggregates older incidents and brute-force results. CISA issued an advisory urging organizations to immediately rotate all Fortinet admin and VPN credentials, enforce MFA, and review VPN session logs from the past 90 days for unauthorized access.

### CVE-2026-42530: Critical NGINX HTTP/3 Use-After-Free (CVSS 9.2) — Patch to 1.31.2 Now
**Source:** The Hacker News / F5 | [Read more](https://thehackernews.com/2026/06/f5-patches-two-critical-nginx-open.html)

F5 released out-of-band patches on June 17 for a critical use-after-free vulnerability (CVE-2026-42530, CVSS v4 9.2) in NGINX's HTTP/3 QUIC module (`ngx_http_v3_module`), affecting versions 1.31.0 and 1.31.1. A remote unauthenticated attacker can send a crafted HTTP/3 session to reopen a QPACK encoder stream, triggering worker-process restarts (DoS) and enabling arbitrary code execution where ASLR is disabled or bypassable. The fix is NGINX Open Source 1.31.2, released June 17; anyone running HTTP/3 should upgrade immediately or disable QUIC by removing `quic` from all `listen` directives as a temporary mitigation. Note that NGINX Ingress Controller and Instance Manager remain unpatched as of the disclosure date — check F5 advisories for their fix timeline.

### PostgreSQL 19 Beta 1: Parallel Autovacuum, Online REPACK, and Property Graph Queries
**Source:** PostgreSQL Global Development Group | [Read more](https://www.postgresql.org/about/news/postgresql-19-beta-1-released-3313/)

PostgreSQL 19 Beta 1 was released on June 4, 2026, introducing several operationally significant features ahead of GA targeted for September/October 2026. Parallel autovacuum lets vacuum workers run concurrently across a table (via `autovacuum_max_parallel_workers`), directly addressing vacuum lag on large high-write tables. A native `REPACK` command enables online table reorganization without a maintenance window, replacing the widely-used `pg_repack` extension for this use case. SQL/PGQ property graph queries implement native graph traversal per SQL:2023, and `INSERT ... ON CONFLICT DO SELECT` adds atomic get-or-create semantics. Breaking changes to watch: JIT compilation is now disabled by default, RADIUS authentication is removed, and `default_toast_compression` defaults to `lz4`.

### EU Cyber Resilience Act: Mandatory Vulnerability Reporting Begins September 11, 2026
**Source:** Hogan Lovells | [Read more](https://www.hoganlovells.com/en/publications/eu-cyber-resilience-act-getting-ready-for-cra-compliance-in-2026)

The EU Cyber Resilience Act enters its first enforcement phase on September 11, 2026, when manufacturers of products with digital elements must begin mandatory vulnerability and incident reporting to ENISA: a 24-hour early warning for actively exploited vulnerabilities and a 72-hour formal notification for severe security incidents. Conformity assessment body provisions are already in effect (since June 11), and full secure-by-design compliance plus CE marking is required by December 11, 2027. The CRA is the world's first horizontal cybersecurity regulation covering the entire software supply chain — including open-source components — with requirements for SBOM documentation, secure update mechanisms, and supplier due diligence. Engineering organizations shipping hardware or software into the EU should treat September as the hard deadline to have incident detection and escalation pipelines operational, even if full product compliance is still 18 months away.
