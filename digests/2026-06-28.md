## Security & Privacy

### DirtyClone (CVE-2026-43503): Working Root Exploit Published, Bypasses File-Integrity Monitoring
_The Hacker News / JFrog Security Research_ | https://thehackernews.com/2026/06/new-dirtyclone-linux-kernel-flaw-lets.html

JFrog Security Research published a working exploit on June 25 for DirtyClone (CVE-2026-43503, CVSS 8.8), a local privilege escalation flaw in the Linux kernel's network packet cloning path where __pskb_copy_fclone() incorrectly drops the SKBFL_SHARED_FRAG safety flag. The exploit gains root by corrupting file-backed page cache memory without writing to disk — on-disk binary hashes remain clean, defeating standard file-integrity monitoring tools. Patch to kernel v7.1-rc5 or apply your distro's backport now; Debian and Fedora are vulnerable in default configurations and Kubernetes cluster nodes with unprivileged user namespaces are especially exposed.

### Amazon Q Developer MCP Auto-Execution Flaw (CVE-2026-12957): From git clone to Cloud Compromise
_Wiz Research / The Register_ | https://www.theregister.com/cyber-crime/2026/06/26/amazon-q-flaw-let-booby-trapped-git-repos-execute-code-swipe-cloud-creds/5263202

Wiz Research disclosed CVE-2026-12957 (CVSS 8.5) on June 26: the Amazon Q Developer IDE extension automatically executed MCP server configurations from .amazonq/mcp.json in cloned repositories without any permission prompt, with spawned processes inheriting the developer's full environment — AWS credentials, CLI tokens, and SSH agent sockets. A single poisoned config file placed in a public repo is sufficient to exfiltrate cloud credentials on git clone. Update to Language Servers for AWS 1.69.0, which adds an explicit trust prompt for untrusted MCP servers before execution.

### Cisco Catalyst SD-WAN CVE-2026-20262: Sixth Exploited Zero-Day of 2026
_SecurityWeek_ | https://www.securityweek.com/cisco-patches-another-sd-wan-zero-day-the-sixth-exploited-in-2026/

Cisco patched CVE-2026-20262 in mid-June, an authenticated remote vulnerability in Cisco Catalyst SD-WAN Manager allowing arbitrary file creation or overwrite on the underlying OS, which directly enables root privilege escalation. This is the sixth Cisco SD-WAN zero-day exploited in the wild in 2026, following CVE-2026-20245 (CLI root escalation via crafted files, exploited since March) and four earlier flaws since January. Six exploited zero-days in a single product in six months represents a sustained, targeted campaign — organizations running unpatched SD-WAN infrastructure should treat this as an active incident, not a routine patch cycle.

## Systems & Engineering

### PostgreSQL 19 Beta 1: SQL/PGQ Property Graphs, ON CONFLICT DO SELECT, Parallel Autovacuum
_PostgreSQL Global Development Group_ | https://www.postgresql.org/about/news/postgresql-19-beta-1-released-3313/

PostgreSQL 19 Beta 1 released June 4 with SQL/PGQ property graph queries bringing native graph traversal into SQL without an external graph database, and ON CONFLICT DO SELECT providing atomic get-or-create semantics — insert and return the new row, or on conflict return the existing one, in a single round trip. Autovacuum gains parallel workers via autovacuum_max_parallel_workers, eliminating single-threaded bottlenecks on large tables, and the release adds REPACK, native JSON output for COPY TO, temporal row-splitting, and in-place partition merge/split. GA is expected September/October 2026; the PostgreSQL team is actively seeking Beta 1 testers to file issues before release candidates.

### DBOS June 2026: RBAC, OpenMetrics, Bulk Workflow Forking, Google ADK Plugin
_DBOS_ | https://www.dbos.dev/blog/new-in-dbos-june-2026

DBOS — the Postgres-native durable workflow platform — shipped role-based access control, an OpenMetrics endpoint, bulk workflow forking, and a Google ADK plugin in June 2026. The architecturally notable change: workflow metadata and application data now coexist in the same Postgres database and update within the same transaction, eliminating the class of distributed consistency bugs where application state and workflow state diverge after a crash. These additions materially advance DBOS as a production platform for teams building durable, transactional AI agent backends on Postgres without a separate workflow database.

### White House Issues AI Executive Order: Voluntary Pre-Release Model Access Program
_Holland & Knight_ | https://www.hklaw.com/en/insights/publications/2026/06/executive-order-on-artificial-intelligence-expands-cybersecurity

A June 2 White House executive order established a voluntary framework under which developers of advanced AI models can provide the federal government with pre-public-release model access for national security and cybersecurity assessments. The order also directs agencies to build an AI cybersecurity clearinghouse, identify AI systems with elevated risk profiles, and strengthen federal AI defenses. While voluntary participation stops short of mandating pre-deployment testing, this is the clearest signal yet of where federal AI oversight is headed — teams building frontier-class models should expect voluntary compliance to become a procurement lever and eventually a regulatory requirement.
