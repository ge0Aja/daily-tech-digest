### OpenAI Launches "Patch the Planet" to Secure Critical Open-Source Software
_OpenAI / Trail of Bits_ | https://openai.com/index/patch-the-planet/

OpenAI launched "Patch the Planet" on June 22 under its new Daybreak initiative, partnering with security firm Trail of Bits to apply AI-assisted vulnerability research to critical open-source software. Trail of Bits dedicated full-time security engineers to run OpenAI's cyber-capable models (GPT-5.5-Cyber and Codex) across 19 initial projects — including cURL, the Go project, Python, Sigstore, NATS, pyca/cryptography, and aiohttp — resulting in 64 pull requests and 51 issues filed in the first week, with many more under coordinated disclosure. The initiative covers the full defensive loop: discovery, validation, severity review, disclosure, patch development, testing, and deployment, with maintainers receiving ChatGPT Pro, Codex Security access, and API credits. Over 30 projects have committed to participate, making this the most scaled AI-driven open-source security effort to date.

### MiniMax M3: First Open-Weight Model With 1M Token Context, Frontier Coding, and Native Computer Use
_MiniMax_ | https://www.minimax.io/blog/minimax-m3

Launched June 1, MiniMax M3 is the first open-weight model to simultaneously deliver frontier-tier coding performance, a 1-million-token context window, and native multimodal capabilities including image, video input, and desktop computer operation. The model uses MiniMax Sparse Attention (MSA), a two-stage architecture that cuts per-token compute at 1M context to 1/20th of the prior generation, yielding over 9x faster prefill and 15x faster decoding. On SWE-Bench Pro, M3 scores 59%, beating GPT-5.5 and Gemini 3.1 Pro, and achieves 70.06% on OSWorld-Verified for computer use tasks. Weights and the full technical report are due on Hugging Face within days; the API is live now with token plans starting at $20/month.

### Meta Llama 4 Scout and Maverick: Open-Weight MoE Models With a 10M Token Context Window
_Meta AI_ | https://ai.meta.com/blog/llama-4-multimodal-intelligence/

Released April 5, Llama 4 Scout (17B active parameters, 16 experts, 109B total) ships with a 10-million-token context window — the largest of any open-weight model — enabling retrieval-free long-document workflows across entire codebases or document corpora without RAG. Maverick (17B active, 128 experts) delivers multimodal reasoning that outperforms GPT-4o and Gemini 2.0 Flash across standard benchmarks, while Scout fits on a single H100 GPU at Int4 quantization. Both are the first open-weight natively multimodal mixture-of-experts models, with weights available on Hugging Face and llama.com. For software engineers, the 10M context window is practically significant: long-horizon coding agents, full-repo analysis, and session-spanning reasoning tasks no longer require chunking workarounds.

_DeepSeek_ | https://huggingface.co/deepseek-ai/DeepSeek-V3.2

DeepSeek V3.2 is a 671-billion-parameter Mixture-of-Experts model released under the MIT license, incorporating DeepSeek Sparse Attention (DSA) and a scalable reinforcement learning framework that drives its high-compute variant to surpass GPT-5 on several benchmarks. ### EU Adopts Cloud and AI Development Act (CADA): Sovereignty Framework Targets 2035 Capacity Tripling
_European Commission_ | https://digital-strategy.ec.europa.eu/en/policies/cloud-and-ai-development-act

The European Commission formally proposed the Cloud and AI Development Act (CADA) on June 3, establishing a tiered four-level assurance framework for cloud services covering data residency, supply-chain control, infrastructure location, and cybersecurity standards. The act aims to triple EU data center capacity by 2035 and reduce strategic dependence on non-EU hyperscalers; key provisions take effect gradually after formal entry into force on August 4, 2026. Engineers deploying AI workloads for European customers need to evaluate which CADA assurance tier applies to their stack, as data residency obligations at the lowest tier are effectively immediate. This represents a fundamental architectural compliance constraint for any team running AI inference, training, or storage pipelines routed through non-EU cloud providers.

## Security & Privacy

### DirtyClone (CVE-2026-43503): Linux Kernel LPE With Working Public Exploit Targets K8s and Multi-Tenant Environments
_SecurityWeek / JFrog_ | https://www.securityweek.com/dirtyclone-linux-kernel-vulnerability-leads-to-root-access/

JFrog Security Research published a full working exploit chain for DirtyClone (CVE-2026-43503, CVSS 8.8) on June 25 — the first public PoC against this high-severity Linux kernel local privilege escalation flaw. The vulnerability exploits a race condition in how the kernel handles reference counting and memory management of cloned socket buffers, allowing any local user to corrupt kernel memory and gain root without leaving traces; it affects Debian, Ubuntu, and Fedora. It is particularly dangerous in multi-tenant cloud environments, Kubernetes clusters, and containerized workloads where a container breakout leads directly to host root. Patch to Linux kernel v7.1-rc5 (commit 9e171fc1d7d7) immediately — the public PoC makes exploitation timelines extremely short.

### Hijacked npm and Go Packages Use VS Code Workspace Tasks to Deploy Python Infostealer
_The Hacker News_ | https://thehackernews.com/2026/06/hijacked-npm-and-go-packages-use-vs.html

Attackers hijacked two npm packages and 16 Go packages, hiding execution inside a VS Code `.vscode/tasks.json` entry configured to run automatically when the project folder is opened — bypassing lifecycle-hook monitoring tools entirely. The payload retrieves encrypted JavaScript via blockchain transaction data, establishes a socket.io backdoor, and deploys a Python infostealer that harvests browser credentials, password managers, cryptocurrency wallets, Git tokens, GitHub CLI configuration, VS Code workspace secrets, and macOS Keychain / Windows Credential Manager data across all three major OSes. The npm packages (uploaded May 25) have been removed; audit your VS Code workspace configurations for unexpected `tasks.json` entries in third-party projects, and use lockfile integrity checks on your full dependency tree.

### AryStinger Botnet Compromises 4,000+ Outdated D-Link Routers, Pivots to NAS Systems
_eSecurity Planet_ | https://www.esecurityplanet.com/weekly-roundup/massive-breaches-ai-risks-and-critical-vulnerabilities-define-this-week-in-cybersecurity-in-june-2026/

The AryStinger botnet exploited legacy vulnerabilities in over 4,000 outdated D-Link routers to convert infected devices into residential proxies, then expanded to target network-attached storage (NAS) systems on the same local network segments. The campaign is a textbook example of how unpatched or end-of-life network equipment — running firmware years past vendor support — functions as a reliable, low-detection initial access vector for botnet operators. For infrastructure and DevOps teams, this is a reminder to include network equipment in formal vulnerability management programs, enforce firmware update SLAs, and isolate NAS systems from direct internet exposure behind a dedicated firewall segment.

### Xsolis Healthcare Breach: 1.4 Million Patient Records Exposed via Phishing
_SharkStriker_ | https://sharkstriker.com/blog/june-2026-data-breaches/

Healthcare AI company Xsolis suffered a phishing-related breach that compromised approximately 1.4 million patient records, adding to an ongoing pattern of healthcare organizations sustaining large-scale credential-based attacks due to the high regulatory and market value of patient data. The breach was traced to a phishing campaign that obtained employee credentials, enabling lateral movement into production systems containing sensitive health information. Healthcare breaches trigger mandatory HHS OCR breach notification obligations and carry substantial HIPAA penalty exposure; for engineers building health-data pipelines, phishing-resistant MFA (passkeys, hardware keys), DMARC/DKIM enforcement, and strict privileged-account segmentation remain the highest-ROI hardening investments against this attack class.
