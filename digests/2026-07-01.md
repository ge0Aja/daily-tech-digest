### Microsoft Launches Seven MAI Models at Build 2026, Including First In-House Reasoning Model
_Microsoft AI_ | https://microsoft.ai/news/building-a-hillclimbing-machine-launching-seven-new-mai-models/

Microsoft unveiled seven new in-house models at Build 2026, with MAI-Thinking-1 as the flagship reasoning model — a sparse Mixture-of-Experts architecture with 35B active parameters, ~1T total parameters, and a 256k-token context window, trained entirely from scratch without distillation from third-party models. It achieves 97.0% on AIME 2025, 94.5% on AIME 2026, and matches Anthropic's Claude on SWE-Bench Pro, while being preferred over Sonnet 4.6 in blind human evaluations. MAI-Code-1, a companion coding model, is available alongside the flagship; the full suite spans text, image, voice, and speech modalities accessible via Azure AI Foundry, Fireworks AI, and Open Router. This marks Microsoft's first credible break from OpenAI model dependency, a significant shift in the AI infrastructure landscape for teams building on Azure.

### Five Eyes Agencies Issue First Joint Agentic AI Security Guidance
_CISA / Five Eyes_ | https://www.theregister.com/2026/05/04/five_eyes_agentic_ai_recommendations/

The Five Eyes alliance (CISA, NSA, Australia's ASD ACSC, Canada's CCCS, New Zealand's NCSC, and the UK's NCSC) published a 30-page joint advisory titled "Careful Adoption of Agentic AI Services" on May 1 — their first coordinated policy document targeting a single AI attack surface. The guidance identifies 5 risk categories (privilege, design/configuration, behavioral, structural, and supply-chain), 23 specific risks, and over 100 best practices, emphasizing that agentic AI amplifies existing organizational security frailties rather than introducing entirely new threat classes. Agencies recommend slow, phased adoption with zero trust, least privilege, and defense-in-depth controls rather than treating agents as a separate security domain. For engineering teams deploying autonomous agents with tool-calling, API access, or production system integration, this document is now the baseline compliance reference from national cybersecurity authorities.

### MIT Research Reduces AI Agent Compute Cost via Sparse Activation and Asynchronous Scheduling
_MIT News_ | https://news.mit.edu/2026/improving-ai-agent-speed-and-energy-efficiency-0625

MIT researchers introduced Murakkab, a system that optimizes the design and deployment of multi-step agentic AI workflows through sparse activation patterns and asynchronous task scheduling, directly reducing both latency and energy consumption. The system eliminates redundant model invocations across sequential tool-calling chains — a common bottleneck in agentic loops — by caching intermediate reasoning states and selectively skipping sub-steps where model confidence already exceeds a threshold. In benchmarks across research, coding, and support agent workflows, Murakkab reduces compute cost by roughly 40% without measurable accuracy degradation. For teams running high-frequency agent workflows at scale, this approach offers a concrete engineering path to cost reduction without model changes.

## Security & Privacy

### Oracle E-Business Suite CVE-2026-46817 Exploited in the Wild Days After First Public PoC
_Help Net Security / Bleeping Computer_ | https://www.helpnetsecurity.com/2026/06/30/oracle-payments-cve-2026-46817-exploitation/

A critical flaw in Oracle E-Business Suite (CVE-2026-46817, CVSS 9.8) — an improper privilege management and authentication bypass in the Oracle Payments File Transmission component — came under active exploitation on June 27, roughly six weeks after Oracle's May 2026 Critical Security Patch Update and before any public proof-of-concept existed. The vulnerability allows unauthenticated HTTP attackers to fully take over Oracle Payments instances on versions 12.2.3 through 12.2.15, with 456 attacks recorded in the first 24 hours of confirmed in-the-wild exploitation. Administrators should apply Oracle's May 2026 CPU immediately; any internet-facing EBS instance unpatched past May 28 should be treated as potentially compromised, with logs reviewed for suspicious POST requests to `/OA_HTML/ibytransmit`. Until patched, EBS web interfaces should be isolated to internal networks only.

### Progress Kemp LoadMaster CVE-2026-8037: Unauthenticated Root RCE via API (CVSS 9.8)
_Security Research_ | https://www.bleepingcomputer.com/news/security/new-oracle-e-business-suite-flaw-now-exploited-in-attacks/

A critical vulnerability in Progress Kemp LoadMaster (CVE-2026-8037, CVSS 9.8) allows an unauthenticated remote attacker to execute arbitrary commands as root on the appliance by sending a single crafted HTTP request to its management API — requiring no credentials, tokens, or prior access. LoadMaster is a widely deployed application delivery controller used for load balancing and SSL offload in enterprise environments, making this a high-value initial access vector for attackers targeting network perimeters. The flaw was disclosed in June 2026 with patches available; organizations running LoadMaster in any internet-exposed or perimeter configuration should treat this as a patch-immediately priority and audit for evidence of unauthorized API requests. Management interfaces for load balancers and ADCs should never be exposed to public networks without MFA and IP allowlisting regardless of patch status.

### Klue Supply Chain Breach: Compromised Legacy Credentials Enable Salesforce CRM Data Access
_BrightDefense / Security Research_ | https://www.brightdefense.com/resources/recent-data-breaches/

Sales intelligence platform Klue confirmed a supply chain breach occurring June 11–12, 2026, in which attackers used compromised legacy credentials to access its integration environment and ultimately reach Salesforce CRM data. The attack vector — stale credentials with live access to production integrations — is a canonical example of the supply chain risk posed by credential sprawl across SaaS ecosystems, where decommissioned accounts or service tokens are not promptly revoked. The incident reinforces the need for automated credential rotation, offboarding checklists tied to integration access, and continuous scanning of integration environments for dormant credentials. Engineering teams building SaaS-to-SaaS integrations should treat OAuth tokens and API keys like infrastructure secrets, with rotation policies and centralized revocation capabilities.

_Google Cloud_ | https://cloud.google.com/blog/products/compute/ironwood-tpus-and-new-axion-based-vms-for-your-ai-workloads

Google's seventh-generation Tensor Processing Unit, codenamed Ironwood (TPU v7), entered general availability on April 22, 2026, delivering 4.6 petaFLOPS per chip and 42.5 exaFLOPS in a 9,216-chip superpod configuration — a 10x peak performance improvement over TPU v5p and 4x better performance-per-chip over TPU v6e (Trillium) for both training and inference. Anthropic committed to up to one million Ironwood chips in 2026 as part of a deal covering over a gigawatt of compute capacity, with the first 400,000 units worth approximately $10 billion in finished racks. Ironwood is available now through Compute Engine, GKE, and Vertex AI; Google is simultaneously previewing an eighth-generation split into dedicated training and inference chips built on TSMC's 2nm process. For teams evaluating large-model training or inference at scale on Google Cloud, Ironwood marks a meaningful cost-per-FLOP improvement over prior TPU generations.

### Kubernetes v1.36 "Haru": User Namespaces GA, HPA Scale-to-Zero Beta, Ingress NGINX Retired
_Kubernetes_ | https://kubernetes.io/blog/2026/04/22/kubernetes-v1-36-release/

Kubernetes v1.36 "Haru" released April 22, 2026, with 70 enhancements: 18 graduating to stable, 25 entering beta, and 25 reaching alpha. The headline GA features are User Namespaces (mapping container root to non-privileged host users, directly hardening container escapes), Mutating Admission Policies using CEL (a native, high-performance alternative to webhook-based mutations), and SELinux Volume Mounting (eliminating costly recursive file relabeling). The long-awaited HPAScaleToZero feature gate, in alpha since v1.16, moves to beta and is enabled by default, allowing HPA to finally scale workloads to zero replicas. Separately, Kubernetes SIG Network and the Security Response Committee retired Ingress NGINX on March 24, 2026, with no further releases, bug fixes, or security updates — teams dependent on it must migrate to a supported ingress controller.

_Tencent / Open Source_ | https://opensource.googleblog.com/2026/

Tencent released TencentDB Agent Memory as an open-source persistent memory system specifically designed for AI agents running long-horizon tasks, addressing the context window limitation that causes agents to lose task state across multi-session workflows. The system provides structured memory storage, retrieval, and compression optimized for agent loop patterns — storing tool call results, observations, and intermediate reasoning in queryable form rather than relying on flat context accumulation. The release also signals that agent infrastructure components are beginning to commoditize as open-source alternatives to proprietary agent memory services.

### Vector Databases Giving Way to Agent-Ready AI-Native Databases as 2026 Workloads Mature
_InfoQ / LinuxInsider_ | https://www.infoq.com/

The consensus among infrastructure teams shipping production AI workloads in 2026 is that single-purpose vector databases are no longer sufficient: agent-ready, AI-native databases that support multiple data types — structured, vector, graph, and time-series — within a unified query engine are becoming the practical target. The driver is that retrieval-augmented generation and multi-step agent workflows require richer context assembly than pure vector search can provide, particularly when combining entity relationships, temporal ordering, and semantic similarity in a single query path. Vendors including SingleStore, Weaviate, and MongoDB Atlas are converging on hybrid query engines that surface all of these within a single SDK surface. For teams choosing a data persistence layer for new AI projects in 2026, pure-vector stores should be treated as a solved but narrow primitive rather than the default architecture.

### Open Source Enters 2026 Under Funding Pressure, Licensing Battles, and Governance Strain
_Linux Insider / Open Source Initiative_ | https://www.linuxinsider.com/story/open-source-in-2026-faces-a-defining-moment-177630.html

The 2026 State of Open Source report documents growing strain across three simultaneous fronts: major projects operating under unsustainable funding models, an increase in proprietary license relabeling disguised as open source (particularly from AI model vendors), and governance fragmentation as enterprise adoption outpaces community capacity. The AI code generation boom is compounding maintenance burden — AI-generated PRs with minimal context are flooding maintainer queues — while the economic value extracted by commercial users continues to vastly outpace financial contributions back to projects. Several high-profile projects are moving to dual licensing or contributor license agreements that restrict cloud provider usage, echoing the HashiCorp and Redis license pivots of 2024. Engineers and organizations consuming open-source dependencies should treat supply-chain sustainability and license compatibility as first-class architecture concerns, not legal afterthoughts.
