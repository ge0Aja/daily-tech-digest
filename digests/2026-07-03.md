### Five Eyes + CISA Publish First Joint Guidance on Agentic AI Security
_CISA_ | https://www.cisa.gov/news-events/news/cisa-us-and-international-partners-release-guide-secure-adoption-agentic-ai

CISA, NSA, and cybersecurity agencies from Australia, Canada, New Zealand, and the UK jointly released the first guidance document focused specifically on security risks in agentic AI systems — a significant signal that real-world deployments have outpaced the security community's response. The document identifies five risk categories: excessive privilege, design and configuration flaws, behavioral drift, structural failure across interconnected agent networks, and supply-chain weaknesses. Core recommendations include cryptographically secured per-agent identities with short-lived credentials, human approval gates for high-impact actions, and applying existing zero-trust and least-privilege principles to agent workflows. The guidance underscores a key concern: most organizations are granting AI agents far more access than they can safely monitor or control.

### AI Moves from Chat to Operating Layer: Agentic Systems Now in Production Across Industries
_AI News_ | https://www.artificialintelligence-news.com/

The dominant story in AI for July 2026 is less about new model launches and more about a structural shift: AI has moved from curiosity to operating layer. Agentic systems — AI that builds plans, uses tools, and completes multi-step tasks autonomously — are now in production across research, software engineering, legal work, customer support, payments, and commerce workflows. For engineering teams, this shift introduces new classes of reliability, observability, and security requirements: agents fail differently than APIs, prompt injection becomes an operational threat, and cost control requires per-agent budgeting. The engineering discipline around agentic system operations is consolidating rapidly, with observability tooling, agent identity management, and approval-gate frameworks emerging as the critical infrastructure layer.

## Security & Privacy

### SharePoint RCE CVE-2026-45659 Hits CISA KEV — Federal Patch Deadline July 4
_Bleeping Computer_ | https://www.bleepingcomputer.com/news/security/cisa-microsoft-sharepoint-rce-flaw-now-actively-exploited/

CISA added CVE-2026-45659 — a CVSS 8.8 remote code execution vulnerability in Microsoft SharePoint Server — to its Known Exploited Vulnerabilities catalog after confirming active in-the-wild exploitation. The flaw arises from deserialization of untrusted data and requires only Site Member-level authentication — no admin rights — to execute arbitrary code remotely on the SharePoint Server. Microsoft had originally rated exploitation as "less likely" when it patched the flaw in May 2026 for SharePoint Server Subscription Edition, 2019, and Enterprise Server 2016. The KEV listing sets a July 4 patch deadline for federal agencies; private organizations should treat the same date as urgent given confirmed active exploitation with unknown threat actors.

### KDDI Breach Exposes 14.2 Million Email Logins Across Six Japanese ISPs
_Bleeping Computer_ | https://www.bleepingcomputer.com/news/security/data-breach-exposes-up-to-142-million-email-logins-at-six-isps/

On June 17, 2026, KDDI detected unauthorized access to a shared email system it operates for six Japanese internet providers — STNet, KDDI Web Communications, JCOM, Chubu Telecommunications, Nifty, and BIGLOBE — exposing login credentials for up to 14.22 million accounts. The root cause was a vulnerability in unnamed third-party software integrated into the shared email platform; KDDI has declined to identify the vendor, making independent risk assessment impossible. Credentials were stored in hashed or encrypted form, but KDDI warns attackers may have extracted them. The incident is a textbook example of shared-infrastructure blast radius: a single third-party software flaw cascaded across multiple independent ISPs, affecting both current and former customers.

### 2026 Threat Landscape: AI-Driven Attacks Surpass 2025 in Severity
_eSecurity Planet_ | https://www.esecurityplanet.com/threats/ai-driven-threats-global-breaches-and-compliance-shifts-define-the-week-in-cybersecurity-for-july-2026/

Security researchers confirm that 2026 has already exceeded 2025 in breach severity, with attackers deploying AI agents for precision phishing, deepfake voice scams, and multi-extortion ransomware at scale. Active groups in the first days of July include LockBit, KRYBIT, SafePay, and INC_RANSOM, with targets spanning healthcare, aviation, and food distribution. Small security teams that rely heavily on chat-based approvals and have high trust in third-party software integrations are the primary victims. Defenders should prioritize hardware-backed MFA, third-party software audits, zero-trust segmentation of agentic workflows, and — critically — human-in-the-loop controls for any AI agent with the ability to take external actions.

### Rust + WebAssembly Crosses to Production in 2026: WASI Preview 2 Stable, Component Model Standardized
_InfoWorld / DevNewsletter_ | https://devnewsletter.com/p/state-of-webassembly-2026/

WebAssembly has moved from experimental to production-grade in 2026, with the Component Model actively standardized, WASI Preview 2 stable, and Preview 3 (async and thread support) as the next milestone. Rust leads the ecosystem via cargo-component tooling, with Wasmtime shipping monthly releases now at version 41. Production deployments include Cloudflare Workers (Rust as a first-class WASM language), Google Sheets' entire calculation engine running on WasmGC, and VS Code's docfind browser-based search engine. In May 2026, Rust 1.96 removed the --allow-undefined flag on WebAssembly targets, aligning WASM platform behavior with native targets — a breaking change for projects relying on the previous behavior.
