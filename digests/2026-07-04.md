### Anthropic Rolls Back Covert Claude Code Tracking Code That Targeted Chinese Users
_Semafor / The Register_ | https://www.theregister.com/ai-and-ml/2026/07/01/anthropic-is-removing-its-covert-code-for-catching-chinese-competitors/5265366

Anthropic removed a hidden tracking mechanism from Claude Code after Semafor reported on July 1 that it had been covertly fingerprinting users based on timezone and suspected affiliation with Chinese AI companies. The technique swapped the apostrophe in "Today's date is" with one of three visually identical Unicode characters depending on which risk flags triggered, along with a date-format change for Chinese timezones — changes invisible to users and even to the model, but machine-parseable by Anthropic's servers. An Anthropic engineer described it as an "experiment launched in March to prevent account abuse from unauthorized resellers and protect against distillation." Alibaba subsequently banned employee use of Claude Code effective July 10, citing the breach of trust, adding to escalating conflict between the two companies over allegations that Alibaba has been training models using Claude's outputs at industrial scale.

## Security & Privacy

### "Bad Epoll" Linux Kernel Flaw (CVE-2026-46242) Gives Any Unprivileged User Root — Hits Android
_The Hacker News_ | https://thehackernews.com/2026/07/new-bad-epoll-linux-kernel-flaw-lets.html

Researcher Jaeyoung Chung disclosed CVE-2026-46242, a use-after-free race condition in the Linux kernel's epoll subsystem where two code paths attempt to free the same kernel object simultaneously, enabling an attacker to corrupt kernel memory and escalate from a normal user account to root. The bug was introduced in kernel v6.4 by a 2023 commit and affects Linux desktops, servers, and Android; a working exploit has been published and the race window is roughly six instructions wide. A patch landed in mainline on April 24, 2026, but distribution rollout is incomplete. Epoll cannot be disabled as it underlies virtually all production server and browser I/O, so patching is the only mitigation — organizations running kernel v6.4+ should treat this as an urgent update.

### SharePoint RCE (CVE-2026-45659) Actively Exploited, CISA Deadline Was Today
_The Hacker News / SecurityWeek_ | https://thehackernews.com/2026/07/sharepoint-rce-cve-2026-45659-added-to.html

CISA added CVE-2026-45659 — a CVSS 8.8 deserialization remote code execution flaw in SharePoint Server 2016, 2019, and Subscription Edition — to its Known Exploited Vulnerabilities catalog on July 1 with a remediation deadline of July 4. Any authenticated attacker with Site Member permissions can exploit the vulnerability remotely to execute arbitrary code on the SharePoint Server without admin rights. Microsoft patched the issue in its May 2026 Patch Tuesday release, but active exploitation is now confirmed and the attack chain requires only standard authenticated access. Organizations running on-premises SharePoint that have not applied the May patches should treat this as an emergency and verify patch status before end of day.

### New Avalon Malware Framework Bundles AI-Assisted Ransomware With Zero AV Detections
_The Hacker News_ | https://thehackernews.com/2026/07/new-avalon-malware-framework-packs.html

Cybersecurity researchers disclosed Avalon, a previously undocumented modular malware framework that combines credential theft, lateral movement, remote access, backup disruption, and ransomware (internally named CrownX) in a single phishing-delivered package. The framework harvests credentials from Chromium and Firefox, cryptocurrency wallets, Discord, Slack, Teams, Windows Credential Manager, SSH known_hosts, and saved RDP connections. Unusually, Avalon shows signs of AI-assisted development and had zero detections across all VirusTotal engines as of its March 2026 upload. Distribution relies on a password-protected archive hosted on Proton Drive delivered via a spoofed legal document email — embedding the payload inside an ISO image to evade email gateway detection — and illustrates how AI tooling is lowering the barrier to building capable, multi-stage malware.

### AirDrop and Android Quick Share: Six Flaws Enable Nearby Unauthenticated Attacks
_Develeap_ | https://www.develeap.com/news/airdrop-and-quick-share-flaws-let-nearby-attackers-trigger-c-bc517925/

Security researchers disclosed six vulnerabilities across Apple's AirDrop and Google's Android Quick Share protocols, enabling nearby attackers to exploit either system without prior authentication. The flaws allow adversaries within wireless range — in offices, airports, coffee shops, or public transit — to trigger malicious interactions on devices with file-sharing enabled. Both Apple and Google have been notified; patches are pending. In the interim, users and IT teams should ensure AirDrop is set to "Contacts Only" or disabled in public environments, and Quick Share should be restricted to known contacts on Android devices.

For infrastructure and platform engineers, the Kilby model — dedicated generation + direct interconnect + 20-year commitment — is likely to become the template for all hyperscale AI expansion over the next decade.

### GitHub Copilot Coding Agent Now Generally Available: Assigns Issues, Opens PRs Autonomously
_GitHub Blog_ | https://github.blog/news-insights/product-news/github-copilot-meet-the-new-coding-agent/

GitHub's Copilot Coding Agent, now generally available, lets developers assign a GitHub Issue directly to Copilot, which then runs autonomously in an isolated GitHub Actions environment — reading the codebase, writing code, running tests and linters, and opening a pull request with a detailed description and task checklist. The agent operates strictly on `copilot/*` branches, cannot touch protected branches, and undergoes a mandatory three-layer scan (CodeQL, secret scanning, dependency review) before the PR surfaces for human review. GitHub positions it for low-to-medium complexity tasks in well-tested codebases: adding features, fixing bugs, extending test coverage, refactoring, and documentation updates. The workflow is accessible from GitHub Issues, VS Code, and the agents panel; teams that have already invested in good issue descriptions and test coverage will see the largest lift.

### FTC Proposes AI Accuracy Policy Statement: Undisclosed Ideological Manipulation Could Violate FTC Act
_FTC.gov_ | https://www.ftc.gov/news-events/news/press-releases/2026/07/ftc-seeks-public-comment-policy-statement-addressing-ai-accuracy

The Federal Trade Commission released a proposed policy statement on July 1 addressing what it calls the "suppression of accuracy" in AI systems, seeking public comment through July 31. The statement holds that AI companies secretly designing their systems to pursue undisclosed ideological objectives — rather than maximizing response accuracy — could violate Section 5 of the FTC Act as unfair or deceptive practices, given that consumers accept AI outputs without independent fact-checking more than 90% of the time. Notably, the FTC singles out Colorado's AI Act as potentially preempted by federal law to the extent it pressures companies to suppress accuracy to avoid disparate impact liability. A safe harbor is proposed for companies that clearly and conspicuously disclose any non-accuracy objectives their AI systems prioritize. For teams building or deploying AI products to US consumers, the comment period is an opportunity to shape the final policy before it becomes an enforcement benchmark.
