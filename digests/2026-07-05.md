### Tencent Open-Sources TencentDB Agent Memory: 4-Tier Local Memory Pipeline for Long-Horizon AI Agents
_MarkTechPost / GitHub_ | https://github.com/TencentCloud/TencentDB-Agent-Memory

Tencent Cloud released TencentDB Agent Memory under the MIT License — a fully local, 4-tier memory pipeline for AI agents requiring zero external API dependencies. The layers progress from L0 Conversation (raw dialogue) through L1 Atom (discrete facts), L2 Scenario (scene blocks), to L3 Persona (persistent user profile), enabling agents to compress and selectively retain context across long-horizon tasks. Benchmarks show 61% reduction in token consumption and a 51% improvement in task pass rates when integrated with the OpenClaw plugin; the default backend is SQLite with sqlite-vec, making it practical for teams shipping agents without external memory services. For engineers building production agents where context length and cost are constraints, this is a concrete open-source alternative to proprietary long-term memory providers.

### Anthropic Publishes Three-Agent Harness Architecture for Multi-Hour Autonomous Coding
_InfoQ / Anthropic_ | https://www.infoq.com/news/2026/04/anthropic-three-agent-harness-ai/

Anthropic published a three-agent harness architecture designed to sustain productive autonomous coding sessions across multiple hours without context window degradation. The design separates responsibilities into a Planner (decomposes a product spec into discrete, self-contained tasks), a Generator (executes one task per fresh context window and produces structured handoff artifacts), and an Evaluator (grades output using predefined criteria, using Playwright MCP to navigate live pages for frontend evaluation). Critically, agents receive hard context resets between roles — rather than compression — with structured artifacts (JSON specs, commit progress, claude-progress.txt) providing continuity. The architecture is directly applicable to teams building agent-based CI/CD and code-review pipelines that currently stall on context limits.

### SharePoint RCE CVE-2026-45659 Added to CISA KEV; Patch Deadline Passed for Federal Agencies
_Bleeping Computer / SecurityWeek_ | https://www.bleepingcomputer.com/news/security/cisa-microsoft-sharepoint-rce-flaw-now-actively-exploited/

CISA added CVE-2026-45659 to its Known Exploited Vulnerabilities catalog following confirmed active exploitation of a deserialization RCE in Microsoft SharePoint Server (CVSS 8.8). The flaw affects SharePoint Subscription Edition, SharePoint Server 2019, and SharePoint Enterprise Server 2016 — patches were shipped in Microsoft's May 2026 Patch Tuesday cycle. The exploitation bar is critically low: any authenticated attacker with Site Member permissions (the lowest non-guest role) can trigger remote code execution without admin rights and without user interaction. U.S. federal civilian agencies had a mandatory patch deadline of July 4; private sector organizations should treat this as an immediate priority, audit for missed May patches, and check for signs of compromise on any unpatched internet-facing SharePoint instances.

_Group-IB / ZeroThreat_ | https://www.group-ib.com/blog/voice-deepfake-scams/

Security researchers are documenting a sharp escalation in AI-augmented attacks: vishing (voice phishing) incidents rose 442% in 2024 and continue accelerating into 2026, with 41% of campaigns now spanning email, voice, and messaging platforms simultaneously. Real-time deepfake voice and video generation is being used to impersonate CFOs and executives mid-call, successfully convincing finance teams to approve fraudulent wire transfers during apparently legitimate video conferences. ### Databricks Lakebase Now GA: AI Agents Build 80–97% of Databases on the Platform
_Forbes / Databricks_ | https://www.databricks.com/company/newsroom/press-releases/databricks-launches-lakebase-new-class-operational-database-ai-apps

Databricks' State of AI Agents report reveals that AI agents now create 80% of all new databases and 97% of test/dev environments on its platform — up from 0.1% just two years ago. The shift was unlocked by Lakebase (now generally available), Databricks' serverless PostgreSQL offering that separates compute from storage, making databases trivial to create, clone, pause, and restart on demand — exactly the access pattern autonomous agents require. At the Data + AI Summit in June, Databricks also unveiled LTAP (Lake Transactional/Analytical Processing), a single open-format copy architecture that lets both transactional and analytical engines operate on the same dataset simultaneously, eliminating the traditional ETL boundary. For teams building agentic applications that need to read and write operational state at scale, Lakebase + LTAP represents the most significant database architecture shift since columnar stores.

### Dropbox Shrinks Backend Monorepo from 87GB to 20GB by Rethinking Git Delta Compression
_InfoQ / Dropbox Tech Blog_ | https://www.infoq.com/news/2026/04/dropbox-reduces-git-optimization/

Dropbox reduced its primary backend monorepo from 87GB to 20GB — and cut clone times from over an hour to under 15 minutes — by rethinking how Git structures object deltas in collaboration with GitHub engineers. The culprit wasn't large binaries or accidental commits: it was Git's default delta compression heuristics, which handle large sets of structurally similar files poorly at scale. The fix involved optimized repacking strategies and tuned delta window and depth parameters, implemented in under 200 lines of configuration-level changes. The result is a 77% reduction in repository size with corresponding improvements to CI fetch performance and developer iteration speed — with no changes to repository content, workflow, or file history. Engineers working on large monorepos experiencing degraded clone and CI performance should examine their Git pack configuration as a first-order intervention before architectural refactoring.

### Trump Signs AI Executive Order: Voluntary 30-Day Pre-Release Review, No Mandatory Framework
_NPR / White House_ | https://www.npr.org/2026/06/02/nx-s1-5844347/ai-safety-trump-executive-order

President Trump signed an executive order on June 2 titled "Promoting Advanced Artificial Intelligence Innovation and Security," establishing a voluntary framework where AI developers can share frontier models with the federal government for testing up to 30 days before public or partner release. The order also directs agencies to create an AI cybersecurity clearinghouse for vulnerability information sharing and establishes benchmarks for assessing AI cyber capabilities. Critically, the framework is entirely opt-in — there is no licensing regime, no preclearance requirement, and no enforcement mechanism for companies that decline. An earlier draft had proposed a mandatory 90-day review window; the final 30-day voluntary version reflects pressure to avoid stifling U.S. competitiveness against China. The practical impact is primarily on frontier model developers, though the cybersecurity clearinghouse mechanism could affect how vulnerability disclosures involving AI systems are handled industry-wide.

### Colorado Repeals and Replaces AI Act Before Enforcement — New Law Takes Effect January 2027
_Crowell & Moring / Norton Rose Fulbright_ | https://www.crowell.com/en/insights/client-alerts/colorado-hits-reset-on-ai-regulation-sb-26-189-repeals-and-reenacts-the-colorado-ai-act

Colorado Governor Jared Polis signed SB 26-189 on May 14, 2026, repealing and replacing the original Colorado AI Act (SB 24-205) before its June 30 enforcement date could arrive. The original law — among the most ambitious state-level AI regulations in the U.S. — required high-risk AI developers to apply a formal duty of care, conduct algorithmic impact assessments, and demonstrate non-discrimination protections. The replacement strips out those substantive obligations, substituting specific disclosure requirements, three-year record-keeping mandates, and a 60-day pre-enforcement cure period, with an effective date of January 1, 2027. The reversal was driven by intense lobbying from the tech industry and reflects the broader difficulty of state-level AI regulation ahead of potential federal preemption. For engineering teams that had been preparing Colorado compliance frameworks, the new requirements are substantially lighter — but the January 2027 date is firm.
