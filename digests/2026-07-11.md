### CISA and Five Eyes Publish "Careful Adoption of Agentic AI Services" Guidance
_CISA / NSA / Five Eyes_ | https://www.cisa.gov/resources-tools/resources/careful-adoption-of-agentic-ai-services

CISA, together with the NSA, ASD ACSC, CCCS, NCSC-NZ, and UK NCSC, published joint guidance on security risks specific to agentic AI systems. The document identifies five risk categories — privilege, design/configuration, behavioral, structural, and supply-chain — with agentic privilege risks highlighted as especially dangerous since a single compromise of an over-privileged agent can cascade across critical infrastructure. Behavioral risks include prompt injection, data poisoning, and agents pursuing goals in unexpected ways through legitimate APIs, generating audit logs that are difficult to parse after the fact. Recommended mitigations follow existing zero trust and least-privilege frameworks, but the guidance explicitly warns against treating agentic systems as ordinary software and urges organizations to fold them into existing governance structures before deployment.

## Security & Privacy

### CVE-2026-46242 "Bad Epoll": Linux Kernel LPE Gives Any Local User Root, Hits Android Too
_The Hacker News / TuxCare_ | https://thehackernews.com/2026/07/new-bad-epoll-linux-kernel-flaw-lets.html

A use-after-free race condition in the Linux kernel's epoll subsystem (CVE-2026-46242) lets any unprivileged local user escalate to root on kernel versions 5.10–6.11, affecting servers, desktops, and Android devices. The exploit requires no capabilities or special permissions — a working public exploit achieves a 99% success rate despite a six-instruction timing window, and there is no way to disable the epoll subsystem as a workaround. The bug traces back to a single 2023 epoll code change; notably, Anthropic's Mythos AI independently discovered a related CVE (CVE-2026-43074) in the same changeset earlier this year. The patch landed upstream on April 24 and was publicly disclosed May 30 — any unpatched production Linux system or Android device remains at risk from local code execution scenarios.

### Cursor IDE "DuneSlide" (CVE-2026-50548 / CVE-2026-50549): Zero-Click RCE via Prompt Injection
_Cato Networks / SecurityWeek_ | https://www.catonetworks.com/blog/duneslide-two-critical-rce-vulnerabilities/

Researchers at Cato AI Labs disclosed two critical (CVSS 9.8) RCE vulnerabilities in Cursor IDE, collectively dubbed DuneSlide, affecting every version before 3.0 — used by more than half of Fortune 500 companies. Both flaws are triggered without user interaction: a malicious payload embedded in an MCP server response or web search result can instruct the LLM to write outside the project sandbox or traverse symlinks to overwrite arbitrary system files, including the cursorsandbox binary itself. Successful exploitation converts sandboxed terminal commands to fully unsandboxed RCE and can compromise connected SaaS workspaces. Cursor 3.0 patches both CVEs; teams should upgrade immediately and audit any MCP server integrations for untrusted content sources.

### Adobe ColdFusion CVE-2026-48282 Actively Exploited Within Two Hours of Disclosure
_Bleeping Computer / CISA_ | https://www.bleepingcomputer.com/news/security/max-severity-adobe-coldfusion-flaw-now-exploited-in-attacks/

A maximum-severity path traversal vulnerability (CVE-2026-48282) in Adobe ColdFusion 2025.9, 2023.20, and earlier is being actively exploited in the wild, with attackers beginning exploitation within two hours of Adobe's patch release. The flaw (CWE-22) allows unauthenticated remote attackers to achieve arbitrary code execution in low-complexity attacks with no user interaction required. CISA added the vulnerability to its KEV catalog on July 7 with a federal remediation deadline of July 10; approximately 800 ColdFusion instances remain internet-exposed according to Shadowserver. Any organization running ColdFusion should patch immediately and check for signs of compromise in server logs, particularly unusual file writes or outbound connections.

_Bleeping Computer / TechCrunch_ | https://www.bleepingcomputer.com/news/security/assuranceamerica-data-breach-exposes-records-of-69-million-drivers/

US insurance company AssuranceAmerica disclosed a breach affecting 6,998,886 customers after an attacker compromised an employee account in March 2026 and copied files containing names, contact information, automobile insurance policy data, driver's license numbers, vehicle information, and claims data. The company detected the intrusion on March 17 but completed its file evaluation only on June 15 — a 115-day gap between detection and notification. The incident underscores continued reliance on single-employee-account compromise as an initial access vector for mass data exfiltration at insurance carriers.

### LLMjacking Evolves: Threat Actors Weaponize Misconfigured Ollama Servers as AI Attack Engines
_Sysdig / The Hacker News_ | https://thehackernews.com/2026/07/threatsday-ai-compute-hijacking-apple.html

Sysdig researchers documented a new evolution of LLMjacking in which threat actors exploit misconfigured Ollama model servers — with no authentication configured — as the reasoning engine for a multi-stage automated offensive security framework dubbed VAPT. Rather than simply draining AI API credits, attackers are now using hijacked model inference to conduct reconnaissance, generate exploits, and orchestrate attacks against downstream targets. This marks a shift from resource theft to using stolen AI compute as a capability multiplier for offensive operations. Organizations running self-hosted LLM servers (Ollama, vLLM, LocalAI) should enforce authentication at the API layer and restrict network exposure to trusted subnets — unauthenticated inference endpoints on public or internal networks represent a dual risk of compute theft and attack infrastructure.

### Microsoft Patches RoguePlanet Defender Flaw Enabling SYSTEM Privilege Escalation
_The Hacker News_ | https://thehackernews.com/2026/07/microsoft-patches-rogueplanet-defender.html

Microsoft patched CVE-2026-50656, a privilege escalation vulnerability in the Microsoft Malware Protection Engine (MpEngine), in version 1.1.26060.3008. The flaw was exploited in ransomware campaigns as BlueHammer (CVE-2026-33825), confirmed by CISA, allowing attackers to escalate from a standard user to SYSTEM on Windows systems running Microsoft Defender. The bug was originally disclosed as a zero-day by an anonymous researcher in April 2026 before its exploitation in ransomware operations was confirmed; CISA has added it to the Known Exploited Vulnerabilities catalog. Defender updates are typically delivered automatically, but administrators of enterprise environments with deferred update policies should verify that engine version 1.1.26060.3008 or later is deployed.

### AI Attack Handoff Time Collapses from 8 Hours to 22 Seconds in Three Years
_eSecurity Planet_ | https://www.esecurityplanet.com/weekly-roundup/ai-driven-attacks-critical-exploits-and-global-breaches-define-this-week-in-july-2026-in-cybersecurity/

Security researchers tracking threat actor coordination report that the time between initial access and handoff to a secondary threat actor — typically a ransomware affiliate or data extortion group — has collapsed from approximately eight hours to 22 seconds over the past three years, driven almost entirely by AI-assisted orchestration. AI is being used to accelerate reconnaissance, credential stuffing, lateral movement scripting, and exfiltration prioritization in a single automated pipeline with near-zero human dwell time between stages. Dutch regulators issued formal warnings on July 10 about the surge in AI-fueled cyberattacks, joining similar advisories from US, UK, and Australian agencies. For security teams, this makes detection-and-response time budgets measured in hours obsolete — automated response capabilities operating in seconds are now the baseline requirement for effective defense.

### AI-Generated Browser Ransomware Abuses Chromium APIs Across Windows, Linux, macOS, and Android
_The Hacker News_ | https://thehackernews.com/2026/07/ai-generated-browser-ransomware-abuses.html

Security researchers disclosed a new class of browser-resident ransomware that abuses Chromium's native file system access APIs to encrypt files directly from the browser context without requiring a traditional malware install. The ransomware code was generated with AI assistance and runs cross-platform — targeting Windows, Linux, macOS, and Android through any Chromium-based browser — by abusing the `showDirectoryPicker` API to request broad filesystem access, which many legitimate web apps already use. Once granted permission, the script encrypts files in place and displays a ransom demand within the browser tab; no executable is written to disk, bypassing most endpoint detection approaches that rely on file-write signatures. Defenders should review browser permission policies in managed environments, restrict `showDirectoryPicker` usage via enterprise browser policies where it is not required by business applications, and ensure browser isolation controls are enforced for untrusted content.
