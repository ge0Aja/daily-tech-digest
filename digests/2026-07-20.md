### GPT-5.6 Closes 30-Year Gap in Convex Optimization with a Single Prompt
**Source:** Northeast Times / Hacker News | [Read more](https://northeasttimes.com/2026/07/19/ai-model-reportedly-cracks-30-year-old-math-problem-with-a-single-prompt/)

A viral Hacker News thread (483 points, 314 comments) describes how OpenAI's GPT-5.6 Sol, guided by a carefully constructed prompt, produced a formal mathematical proof closing a gap in convex optimization theory open for roughly 30 years — proving that optimization over a standard function class requires Ω(d²) function evaluations, matching the upper bound of an algorithm published three decades ago. The result remains unverified at press time: skeptics note that LLMs cannot reliably distinguish correct from incorrect proofs, and one analysis argues the output likely reflects prior human research injected via the prompt rather than autonomous mathematical discovery. Regardless of attribution, the episode illustrates that AI-assisted mathematical research is now producing results that draw serious scrutiny from professional mathematicians — a threshold crossed just months ago.

### GitHub Cuts Agentic CI Token Spend Up to 62% with MCP Pruning and Daily Auditors
**Source:** GitHub Blog / InfoQ | [Read more](https://github.blog/ai-and-ml/github-copilot/improving-token-efficiency-in-github-agentic-workflows/)

GitHub's platform team published a detailed breakdown of how they cut per-workflow token costs up to 62% across production agentic CI pipelines. The core finding: a 40-tool MCP server adds 10–15 KB of tool-schema context per API call — pruning unused tools to match each workflow's actual requirements cut 8–12 KB per turn. Supplemental gains came from replacing MCP calls with `gh` CLI pre-fetches for PR diffs and file contents, and from running daily "auditor" and "optimizer" agents that surface expensive workflows and propose code-level fixes. The team routes all agent calls through an API proxy and writes a `token-usage.jsonl` artifact per run using an Effective Tokens metric (output weighted 4×, cache reads 0.1×, with model-tier multipliers). Any team running agentic CI should audit MCP server footprint before assuming token costs are irreducible.

## Systems & Engineering

### TypeScript 7.0 GA: Go-Native Compiler Delivers ~12x Faster Builds — No API Yet for Vue, Svelte, Angular
**Source:** The Register / InfoWorld | [Read more](https://www.theregister.com/devops/2026/07/09/speedier_type_checks_in_typescript_70_as_first_stable_go_release_ships/5268828)

TypeScript 7.0 reached general availability on July 8, completing "Project Corsa" — a port of the compiler from TypeScript-on-Node.js to native Go, preserving identical type-checking semantics without redesigning the architecture. Microsoft's VS Code codebase (2.3M lines) now builds in 10.6 seconds versus 125 seconds under TypeScript 6, an 11.9× improvement; typical project gains run 8–12×. The release ships without a stable programmatic API, which blocks Vue, Angular template type-checking, Svelte, Astro, MDX, and Volar-based tooling from adopting 7.0 — TypeScript 7.1, expected around October 2026, will deliver the new API. Teams using plain `tsc` or ts-node can migrate immediately; those dependent on framework toolchain integration should wait for 7.1.

### Linux 7.2-rc4: Cache-Aware Scheduling, BPF Hardening, strncpy Finally Retired
**Source:** Linux Compatible / Phoronix | [Read more](https://www.linuxcompatible.org/story/linux-kernel-72rc4-drops-cacheaware-scheduling-mongodb-speedups-and-the-end-of-strncpy)

Linus Torvalds tagged Linux 7.2-rc4 on July 19, pushing the codebase past 43 million lines ahead of a stable release expected in August. Headline additions: Cache-Aware Scheduling (CAS), opt-in via `CONFIG_SCHED_CACHE` and tunable through debugfs, keeps cache-sharing tasks within the same LLC domain — benchmarks show up to 100% MongoDB throughput gains on AMD Zen 5 topologies. BPF gains hardening against JIT spraying attacks. The six-year `strncpy` removal campaign closes permanently. Early NOVA Rust GPU driver work for Hopper and Blackwell GPUs lands in the tree. The RISC-V 64-bit core limit doubles from 64 to 256 to match current hardware. Database operators on modern AMD hardware should track the 7.2 stable release date for the CAS gains alone.

## Industry & Trends

### EU AI Act Full Enforcement Begins August 2 — €35M Fines, 13 Days Out
**Source:** Legalnodes / Holland & Knight | [Read more](https://www.legalnodes.com/article/eu-ai-act-2026-updates-compliance-requirements-and-business-risks)

The EU AI Act becomes fully enforceable on August 2, 2026, activating high-risk AI system obligations: risk management frameworks, technical documentation, conformity assessments, EU database registration, post-market monitoring, and transparency labeling for synthetic content. Penalty structure: up to €35M or 7% of global annual turnover for prohibited-AI violations, up to €15M or 3% for high-risk non-compliance. A proposed "Digital Omnibus" extension pushing Annex III obligations to December 2027 has not been finalized — treat August 2 as the binding deadline. Teams shipping AI products into EU markets should confirm conformity assessments are complete, incident-reporting pipelines are wired up, and synthetic-content disclosure is implemented before the deadline.
